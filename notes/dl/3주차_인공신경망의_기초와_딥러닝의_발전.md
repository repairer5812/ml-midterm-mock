# 🚀 [딥러닝 3주차] 인공신경망의 기초와 딥러닝의 발전

## 1. 🌱 인공신경망의 태동: Perceptron부터 Feed-forward NN까지

### 1.1 선형 함수에서 비선형 함수로 (Basis Function)
- 기본 개념: 입력 $x$에 대해서는 비선형 함수가 되도록 Feature Transformation을 수행합니다.
- Basis Function의 선택: 사용자가 미리 정해놓고 사용할 수 있습니다. (예: 푸리에 변환, 푸리에 베이시스 등)
- 핵심 아이디어: 하지만 개념을 미리 정해놓지 않고 데이터로부터 이 함수를 찾았으면 좋겠다(Data-driven)는 아이디어가 바로 Feed-forward Neural Network(FNN)의 핵심 요인 중 하나입니다.

### 💡 [핵심 출제 포인트] SVM vs. Neural Network의 차이점
딥러닝이 뜨기 전 많이 사용되었던 SVM(Support Vector Machine)과 Neural Network는 Basis Function을 정의하는 데 있어 완전히 다른 방향성을 가집니다.

### 🛡️ SVM (Support Vector Machine)
- 중요 샘플 추출: 원래 가졌던 데이터셋 본체에서 풀고자 하는 문제의 가장 핵심이 되는 샘플들(Support Vectors)을 찾습니다.
- 유사도 비교 (Kernel): 새로운 입력이 들어오면 이 Support Vector들과의 유사도를 비교하여 Basis Function을 정의합니다.
- 특징: 중요한 샘플이 무엇인지는 학습에 맡기지만, 유사도를 계산하는 함수(Kernel, 예: RBF 커널 등)는 사용자가 미리 정의해 놓습니다.

### 🧠 Neural Network (NN)
- Data-driven: Basis Function의 함수적 형태(Functional form)는 정해놓지만, 그 폼 안에 있는 파라미터를 데이터로부터 학습합니다. (함수 자체를 학습 가능한 형태로 만듦)

## 2. 🔬 생물학적 뉴런의 모델링
뉴럴 네트워크는 기본적으로 생물학적 신경망(Biological Neural Network)에서 시작되었습니다.

- Dendrite (수상돌기): 수용체로 입력을 받아들이는 부분. 여러 개가 있어 여러 입력을 받아들입니다. (모델링: 입력 $x$)
- Soma (세포체): 들어온 입력을 취합하는 부분. (모델링: 가중치를 곱해 더하는 Weighted Sum)
- Axon (축삭돌기): 취합된 신호를 밖으로 전달하는 부분.
- Synapse (시냅스): 뉴런과 뉴런이 붙어있는 연결 부위. (모델링: 가중치 $w$)
- Activation Function (활성화 함수): Soma가 결정을 해서 다음 뉴런을 활성화(Excite) 시킬지, 억제(Inhibit) 시킬지, 신호를 죽일지 살릴지를 판단하는 함수입니다.

## 3. 🚪 Perceptron (퍼셉트론, 1958)
1958년 Rosenblatt에 의해 처음 제안된 가장 기본적인 계산 모델입니다.

- 구조: 입력 $x$와 가중치 $w$를 선형 결합(Weighted Sum)한 뒤, 활성화 함수(Step function)를 통과시켜 출력을 냅니다.
- Classification (분류): Target은 +1(Positive) 또는 -1(Negative)로 둡니다.
- 학습 방식 (Stochastic Gradient Descent 개념의 시초): 모든 샘플을 한 번에 계산하지 못하고, 샘플을 하나씩 보면서 맞으면 넘어가고 틀리면 업데이트하는 방식을 취했습니다.

### 💡 교수님의 기하학적 예시 (매우 지엽적 출제 가능성)
- $w$는 선형 경계의 기울기를 결정하는 법선 벡터(Normal Vector) 역할을 합니다.
- 초기 $w$(검은색 벡터)로 분류를 테스트했을 때, 파란색 영역에 있어야 할 빨간 점이 틀리게 분류되었다면?
- 틀린 샘플의 벡터 크기만큼 원래 법선 벡터에 더해서 $w$의 방향을 바꿉니다.
    $$
    w^{(new)} = w^{(old)} + \Delta w
    $$
- 모든 샘플이 다 분류될 때까지 이 과정을 반복합니다.
- 한계점 (파멸적 문제): 선형으로 분리 가능한(Linearly separable) 문제는 반드시 답을 찾아내지만, 선형 분리가 불가능한 경우에는 영원히 답을 찾지 못하고 무한 루프에 빠집니다. (이를 해결하기 위해 Multi-Layer Perceptron이 등장하기까지 수십 년이 걸렸습니다.)

## 4. 📈 Activation Function (활성화 함수)의 역사적 변천사
교수님께서 연대별로 활성화 함수의 변화를 매우 강조하셨습니다.

- 1960년대 (초창기): Step Function (계단형 함수)
    - 문제점: 미분이 불가능하여 수학적 최적화(Gradient Descent 등)를 적용하기 매우 어려웠습니다.
- 1980년대: Sigmoid, Tanh (Hyperbolic Tangent)
    - 스텝 함수를 시그모이드나 Tanh로 바꾸는 데 20년이 걸렸습니다.
    - 💡 왜 많이 썼는가? (핵심 장점): 단순히 미분이 잘 되는 것을 넘어, 미분했을 때 원래 자기 자신의 함수 꼴로 표현이 가능하여 계산을 다시 할 필요가 없는 엄청난 장점이 있었습니다.
    (예: $f'(a) = f(a)(1-f(a))$)
- 2010년대: ReLU (Rectified Linear Unit)
    - 데이터가 많아지며 딥러닝이 부흥할 때 획기적인 변화를 이끌었습니다. (Sigmoid/Tanh의 치명적 단점인 Vanishing Gradient 완화)
    - 현재는 ReLU 및 그 변형들(Leaky ReLU, Parametric ReLU 등)이 사실상 표준으로 정해져서 사용됩니다.

## 5. 🥞 Multi-Layer Perceptron (MLP)와 Hidden Layer
- 네트워크 구조의 확장: 입력(Input)과 출력(Output) 사이에 여러 층의 Hidden Layer(은닉층)를 쌓아 올립니다. 각 유닛들이 하나의 퍼셉트론 역할을 하면서, 동시에 그 출력이 다음 층의 Basis Function 입력이 되는 복합 함수(Composite Function) 구조입니다.
- 비선형 문제 해결: Layer를 쌓고 비선형 활성화 함수를 사용함으로써, 선형 모델이 풀지 못하던 복잡한 비선형 문제를 풀 수 있는 강력한 Capacity를 갖게 되었습니다.

### 💡 왜 'Hidden(은닉)' 레이어라고 부르는가? (교수님 강조 포인트)
- 출력층(Output Layer)은 Ground Truth(정답 Target)가 존재하므로, 출력값과 정답을 비교하여 Error(Loss)를 계산하고 평가(Evaluation)할 수 있습니다.
- 하지만 중간에 있는 레이어들은 정답(Target)을 모릅니다. 즉, 얘네가 잘하고 있는지 평가할 수단이 없고 에러에 대한 개념이 가려져(숨겨져) 있기 때문에 Hidden Layer라고 부릅니다. (이 문제를 해결하기 위해 도입된 것이 Backpropagation입니다.)

## 6. 🎯 Network Training과 Error Function (학습과 에러 함수)
히든 레이어는 정답을 모른다고 했습니다. 그렇다면 학습은 어떻게 이루어질까요? 학습의 기준점은 출력층(Output Layer)에서 발생하는 에러(Error)입니다.

- 출력층의 세팅 (풀고자 하는 문제에 따라 다름):
    - Regression (회귀): Continuous output을 원할 때는 출력층의 활성화 함수로 Identity Function(입력을 그대로 출력)을 사용합니다. 에러 함수로는 Sum-of-Squares Error (MSE 등)를 사용합니다.
    - Classification (분류): 확률값으로 바꾸기 위해 출력층에 Sigmoid (이진 분류) 또는 Softmax (다중 분류)를 사용합니다. 에러 함수로는 Cross-Entropy Error를 사용합니다.
- 학습의 목표: 출력값 $y$와 내가 원하는 정답(Target, Ground Truth) $t$의 차이를 계산하여, 이 에러(Loss)를 최소화(Minimize)하는 파라미터 $W$를 찾는 것입니다.

## 7. 🧮 Parameter Optimization & Gradient Descent (파라미터 최적화)
수많은 가중치($W$)들을 어떻게 조절해야 에러가 줄어들까요? 교수님께서 파라미터 공간(Weight Space)에서의 기하학적 해석을 매우 강조하셨습니다.

- 기하학적 해석:
    - $W_1, W_2$를 축으로 하는 파라미터 공간이 있다고 상상해 봅시다. 수직축은 에러(Error) 값입니다.
    - 초기에는 아무 랜덤한 값($W$)에 위치시킵니다. 그 지점에서 출력값과 정답을 비교해 에러를 계산합니다.
- 미분(Gradient)의 의미:
    - 해당 지점에서 미분을 취합니다 (편미분을 모은 Gradient Vector).
    - 💡 [핵심 출제 포인트] 미분 방향의 진짜 의미: 미분값은 그 지점에서 함수값(에러)이 '가장 크게 커지는(증가하는) 방향'을 알려줍니다.
    - 하지만 우리의 목표는 에러를 '줄이는' 것입니다. 따라서 미분값이 가리키는 방향의 반대 방향(Negative Gradient, 마이너스를 붙임)으로 이동해야 합니다. 이것이 바로 Gradient Descent (경사하강법)입니다.
    - (참고: 만약 에러 함수가 아니라 목적/효용 함수(Utility Function)를 최대화하는 문제라면 마이너스를 붙이지 않고 Gradient 방향 그대로 이동합니다. 이를 Gradient Ascent라고 합니다.)

### 🏃 Learning Rate (학습률, $\eta$)
- 구한 Gradient 정보를 100% 그대로 반영하여 이동하지 않습니다. 보통은 아주 작은 값(예: 0.001)으로 스케일링 다운시켜 조금씩 이동합니다.
- $\eta$가 너무 클 때: 답을 빨리 찾을 것 같지만, 실제로는 답 주변에서 진동하거나 오히려 답에서 점점 멀어지는 발산(Divergence) 현상이 발생합니다.
- $\eta$가 너무 작을 때: 학습이 너무 느려집니다.

### 🗣️ [Q&A 내용 포함]
- Q. 학습률이 작다/크다의 기준은 계산에 의한 것인가요, 경험적인가요?
    - A. 거의 실험적, 경험적으로 결정됩니다. (보통 $10^{-5}$ 등에서 시작). 나중에는 처음에 크게 움직이고 점차 줄여가는 Learning Rate Scheduling 기법을 많이 씁니다.

## 8. 📦 Data Batching Strategies (데이터 학습 단위)
데이터를 한 번에 얼마나 보고 Gradient를 계산할 것인가에 대한 3가지 전략입니다. 각 전략의 장단점과 Local Optima 탈출 여부가 시험 단골 출제 포인트입니다.

1. Batch (배치):
    - 개념: 가진 모든 데이터를 다 계산한 후, 평균적인 에러를 가지고 딱 1번의 Gradient를 만들어 업데이트합니다.
    - 장점: 노이즈가 상쇄되어 Gradient 방향의 신뢰성이 매우 높고 부드럽게(Smooth) 이동합니다.
    - 단점: Local Optima(지역 최적해)에 빠지면 절대 빠져나올 방법이 없습니다. (모든 데이터가 동의해서 온 길이라 다른 방향으로 튈 여지가 없음)
2. Online / Stochastic (온라인 / 확률적 경사하강법, SGD):
    - 개념: 데이터 샘플 1개만 보고 바로 Gradient를 계산해 업데이트합니다. (100개의 데이터면 100번 업데이트)
    - 장점: 샘플마다 노이즈가 끼어 있어 방향이 매번 달라지므로, Local Optima에 빠졌을 때 튕겨져 나올 여지가 있습니다.
    - 단점: 1개만 보고 판단하므로 신뢰성이 떨어지고 궤적이 매우 불안정합니다.
    - 💡 [매우 중요한 디테일] 왜 'Stochastic(확률적)'이라고 부르며, Shuffling(섞기)이 왜 필수인가?:
        - 만약 1번부터 100번까지 순서대로만 계속 학습시키면, 신경망이 데이터의 패턴이 아니라 "1번 다음엔 2번이 온다"는 순서 자체를 외워버립니다. (특정 궤적 패턴에 고착됨)
        - 이를 방지하기 위해 매 에포크마다 데이터를 셔플링(Shuffling)하여 랜덤성을 부여합니다. 이 때문에 'Stochastic'이라는 용어를 씁니다.
3. Mini-Batch (미니 배치):
    - 개념: Batch와 Stochastic의 Trade-off(절충안). 전체 데이터를 작은 그룹(예: 10개씩)으로 쪼개서 그룹 단위로 업데이트합니다.
    - 장점: 신뢰성도 어느 정도 확보하면서, 업데이트 횟수도 늘리고, Local Optima도 빠져나올 수 있어 현재 딥러닝에서 가장 많이 쓰이는 표준 방식입니다.

## 9. 🔙 Backpropagation (오차 역전파 알고리즘)
앞서 히든 레이어는 정답을 몰라 에러를 계산할 수 없다고 했습니다. 이를 해결한 전설적인 알고리즘입니다.

- 개념 (책임 전가): 출력층에서 발생한 에러를 10이라고 할 때, 이 에러를 만든 이전 유닛들에게 "너희들이 연결된 가중치(연결 강도)만큼 책임이 있으니 에러를 나눠 가져라"라며 거꾸로(Backward) 에러를 물어주는 방식입니다.
- 수식적 원리 (Chain Rule):
    - 파라미터 $W$에 대해 미분할 때, 합성함수의 미분법인 Chain Rule(연쇄 법칙)을 사용합니다.
        $$
        \frac{\partial E}{\partial W} = \frac{\partial E}{\partial A} \times \frac{\partial A}{\partial W}
        $$
    - 교수님 비유: 분자, 분모에 같은 항목이 있어 약분되면서 지워지는 효과가 발생하여, 결국 끝(Error)에서부터 시작점(Weight)까지 미분값이 쫙 곱해지면서 전달됩니다.
    - 각 레이어를 거칠 때마다 활성화 함수의 미분값($f'(a)$)이 계속해서 곱해집니다. (이것이 나중에 큰 문제인 Vanishing Gradient를 유발하게 됩니다.)

## 10. 🕸️ Computational Graph (연산 그래프와 자동 미분)
교수님께서 "지금까지 온 것 중 가장 큰 폭발력 중 하나가 이 새로운 획기적인 개념"이라고 극찬하신 부분입니다.

- 과거의 고통 (Manual Derivation):
    - 예전에는 사용자가 활성화 함수를 바꾸거나 레이어 구조를 조금만 변경해도, 연구자가 일일이 미분 공식을 손으로 계산(Derive)해서 하드코딩해야 했습니다.
    - 수식이 조금만 틀려도 버그를 잡기가 불가능에 가까웠고, 이 때문에 다양한 구조를 테스트하기가 너무 어려웠습니다.
- Computational Graph의 도입 (PyTorch, TensorFlow 등):
    - 네트워크 구조(예: $x$와 $w$를 곱해서 $u$에 넣고, $b$를 더해서…)를 코드로 정의하기만 하면, 프레임워크 내부(Behind the scenes)에서 이를 노드(연산)와 엣지(데이터)로 이루어진 그래프(Graph)로 쫙 바꿔버립니다.
- Auto-Differentiation (자동 미분):
    - 그래프의 출력단에서부터 거꾸로 내려오며(Backprop 방향) 각 노드에 정의된 미분 규칙을 통해 미분값이 자동으로 계산되어 튀어나옵니다. 사용자는 더 이상 손으로 미분할 필요가 없어졌습니다!

## 11. 🌌 Universal Approximation Theorem과 Deep Architecture
- Universal Approximation Theorem (보편적 근사 정리):
    - 이론적으로는 단 하나의 Hidden Layer(은닉층)만 있어도, 유닛(노드)의 개수가 충분히(어마어마하게) 많다면 이 세상의 어떤 연속 함수(Continuous function)든 모두 표현할 수 있습니다.
    - 문제점: 유닛 개수가 늘어날수록 Fully Connected 구조에서는 파라미터 개수가 기하급수적으로 증가하고, 이를 학습하기 위한 데이터도 기하급수적으로 필요해집니다. 결국 현실적으로는 불가능에 가깝습니다.
- Deep Architecture의 핵심 아이디어 (2009년 벤지오 교수):
    - 한 번에 복잡한 패턴을 다 배우려고 얕고 넓게(Shallow & Wide) 만들지 말고, 심플한 것들을 계층적으로 누적해서 복잡한 것을 만들자(Deep)는 개념입니다.
- 💡 교수님의 얼굴 인식 예시 (계층적 학습):
    - Low-level: 픽셀들 간의 가장 심플한 관계 (선, 엣지 등)
    - Intermediate-level: 선들을 결합해 코너, 교차점(Cross) 등 모티프(Motif) 생성
    - High-level: 모티프를 결합해 눈, 코, 입 생성 $\rightarrow$ 최종적으로 이들을 조립해 얼굴(Face) 인식
- 생물학적 타당성: 실제 인간의 뇌(Brain)도 시각 피질(Visual Cortex) V1, V2, V4를 거치며 점진적이고 계층적으로 정보를 처리합니다.

## 12. ⚠️ 딥러닝의 3가지 난제와 Vanishing Gradient (기울기 소실)
MLP 개념이 나왔을 때부터 레이어를 깊게 쌓고 싶었지만, 2010년대 이전까지는 불가능했습니다. 그 이유는 3가지입니다.

1. 데이터 부족 (Lack of Data)
2. 컴퓨팅 파워 부족 (Lack of HW): 당시에는 CPU만 썼고 GPU 연산이 없었습니다.
3. 💡 [핵심 출제 포인트] Vanishing Gradient (기울기 소실) 문제:
    - Backpropagation은 출력층에서부터 입력층으로 미분값을 계속 '곱하면서' 전달합니다.
    - 당시 쓰던 Sigmoid의 최대 미분값은 0.25, Tanh는 1.0입니다. 즉, 대부분의 미분값이 1보다 작습니다.
    - 1보다 작은 값들을 레이어를 거치며 계속 곱하다 보니, 입력단(Input layer) 근처에 가면 Gradient(에러 신호)가 거의 0에 가까워집니다.
    - 교수님의 찰떡 비유 ("Garbage in, Garbage out"): 입력단에서는 학습이 안 되어 가비지(Garbage)를 뒤로 넘기는데, 뒷단(출력단 근처)에서 아무리 그 가비지를 최적화해봤자 결국 나오는 건 가비지일 뿐입니다. 즉, 전체 학습이 무의미해집니다.

## 13. 🛠️ 한계 극복 1: Greedy Layer-wise Pre-training과 Auto-Encoder
2006년 제프리 힌튼 교수가 제안하여 딥러닝의 부흥을 이끈 전설적인 방법입니다. 한 번에 전체를 학습시키려니 기울기가 소실되니, "레이어를 잘라서 부분부분 먼저 학습시키자"는 전략입니다.

- Auto-Encoder (오토인코더 / Auto-Associator):
    - 입력(Input) $\rightarrow$ 히든(Hidden) $\rightarrow$ 출력(Output)으로 구성된 2-Layer 네트워크입니다.
    - 💡 특징: 출력층의 타겟(Target)을 '입력값 자기 자신'으로 둡니다. 즉, 입력 데이터를 히든 레이어(더 작은 차원)로 압축(암호화)했다가, 다시 원래 데이터로 복원(복호화)하는 능력을 학습합니다.
- Pre-training (사전 학습) 과정:
    1. 원본 데이터로 첫 번째 오토인코더를 학습시킵니다. 학습이 끝나면 출력층(디코더 부분)은 버리고, 히든 레이어의 가중치($W_1$)만 고정(Fix)합니다.
    2. 고정된 첫 번째 히든 레이어의 출력값을 입력으로 삼아, 두 번째 오토인코더를 학습시킵니다.
    3. 이 과정을 원하는 깊이만큼 반복합니다.
- Fine-Tuning (미세 조정):
    - 이렇게 층층이 똑똑해진(유의미한 초기값을 가진) 가중치들을 쭉 쌓아 올린 뒤, 마지막에 Classification 층을 붙여 전체 네트워크를 한 번에 학습시킵니다.
    - 이미 좋은 출발점(초기값)에 가 있기 때문에 Vanishing Gradient에 빠지지 않고 학습이 잘 됩니다.
    - (Q&A 내용: 전이 학습(Transfer Learning)도 이와 유사한 원리이나, 학습 데이터와 새 데이터의 분포(Distribution)가 어느 정도 유사해야만 효과가 있습니다.)

## 14. 🛠️ 한계 극복 2: ReLU 활성화 함수의 도입
- ReLU (Rectified Linear Unit):
    - 음수는 0으로 죽여버리지만, 양수 구간에서는 기울기(미분값)가 항상 1입니다.
    - 1을 아무리 여러 번 곱해도 작아지지 않으므로, Vanishing Gradient 문제를 수학적으로 아주 심플하고 완벽하게 완화했습니다.
- Leaky ReLU 등 변형의 등장 이유:
    - ReLU는 음수일 때 미분값이 0이 되어 뉴런이 아예 죽어버리는(Dead ReLU) 현상이 발생합니다.
    - "내가 이 뉴런이 나중에 필요할 수도 있지 않겠느냐?"라는 아이디어에서, 음수 구간에도 아주 작은 기울기(예: 0.01)를 살려두는 Leaky ReLU, Parametric ReLU(기울기마저 학습) 등이 등장했습니다.

## 15. 🧰 딥러닝 필수 테크닉 (Tricks of the Trade)
교수님께서 지엽적으로 출제하기 아주 좋은 디테일한 기법들입니다.

- Early Stopping (조기 종료):
    - 학습 곡선(Learning Curve)을 그릴 때, Training Error가 계속 줄어든다고 해피(Happy)해하면 안 됩니다. 오버피팅(Overfitting)이 발생할 수 있습니다.
    - 반드시 Validation Set(검증셋)의 에러를 함께 확인하여, 검증 에러가 다시 올라가기 시작하는 지점에서 학습을 강제로 멈춰야 합니다.
- Denoising Auto-Encoder (디노이징 오토인코더):
    - 원본 입력에 일부러 노이즈(Noise)를 더하거나 일부를 지워버립니다. (예: 이미지에 검은 점 추가)
    - 하지만 복원해야 하는 타겟(Target)은 노이즈가 낀 데이터가 아니라 '노이즈가 없는 원본 데이터'입니다.
    - 이를 통해 불필요한 노이즈를 스스로 걸러내는 훨씬 Robust(강건)한 피처를 학습하게 됩니다.
- 💡 [핵심 출제 포인트] Dropout (드롭아웃):
    - 네트워크를 그냥 학습시키면, 서로 다른 뉴런들이 비슷한 특징을 중복해서 학습하는 Co-adaptation(동조화) 현상이 발생해 리소스를 낭비합니다.
    - 이를 막기 위해 학습 과정에서 임의의 순간에 랜덤으로 뉴런(노드)들을 지워버립니다(존재하지 않는 것처럼 가림).
    - (참고: 노드 대신 연결선(Connection/Weight) 자체를 지워버리는 기법은 DropConnect라고 합니다.)
- Batch Normalization (배치 정규화):
    - Internal Covariate Shift: 레이어를 통과할 때마다 출력값들의 분산이 너무 커져서(왔다 갔다 해서) 다음 레이어가 학습하기 힘들어하는 현상입니다.
    - 해결책: 미니 배치 단위로 출력값들을 강제로 정규화(평균 0, 분산 1)시켜서 다음으로 넘깁니다.
    - 단, 강제로 정규화하면 신경망의 표현력(Capacity)이 떨어질 수 있으므로, 이를 다시 복구하기 위해 Scale($\gamma$)과 Shift($\beta$)라는 학습 가능한 파라미터를 추가합니다.

## 16. 🧭 방향성을 개선하는 방법 (Momentum 계열)

### 1) Momentum (모멘텀, 관성)
- 개념: 과거에 Gradient가 이동해 온 '역사(History)와 관성'을 현재 업데이트에 반영하자는 아이디어입니다.
- 원리: 이전 스텝들의 업데이트 벡터를 일정 비율($\gamma$, 보통 0.9)로 누적하여 현재 Gradient에 더합니다.
    $$
    v_t = \gamma v_{t-1} + \eta \nabla_w f(w)
    $$
    $$
    w = w - v_t
    $$
- 효과:
    - Gradient가 계속 같은 방향을 가리키는 차원에서는 가속도(Accelerate)가 붙어 더 빨리 수렴합니다.
    - Gradient 방향이 자꾸 바뀌는(진동하는) 차원에서는 지그재그 움직임(Oscillation)을 상쇄시켜 부드럽게 이동하게 해줍니다.

### 2) NAG (Nesterov Accelerated Gradient)
- 개념: 모멘텀의 업그레이드 버전으로, "미리 앞을 내다보자(Look ahead)"는 아이디어입니다.
- 💡 [핵심 출제 포인트] Momentum과 NAG의 결정적 차이점:
    - Momentum: 현재 위치에서 Gradient를 먼저 계산하고, 그 값과 기존의 관성을 더해 크게 점프합니다.
    - NAG: 기존의 관성 방향으로 일단 먼저 크게 점프해 본 다음($w - \gamma v_{t-1}$), 그 '미래의 예상 위치'에서 Gradient를 측정하여 방향을 교정(Correction)합니다. 이로 인해 불필요하게 궤도를 벗어나는 것을 방지합니다.

## 17. 📏 스텝 사이즈(학습률)를 조절하는 방법 (Ada 계열)
단순 SGD는 모든 차원(파라미터)에 똑같은 $\eta$(학습률)를 곱합니다. 하지만 파라미터마다 다른 Learning Rate를 적용하면 더 좋은 답을 찾을 수 있습니다.

### 1) Adagrad (Adaptive Gradient)
- 개념: 파라미터별로 학습률을 다르게 적용합니다.
- 원리: 자주 등장하여 많이 업데이트된 파라미터는 학습률을 작게 줄이고, 드물게 등장한 파라미터는 학습률을 크게 유지합니다. (Sparse data에 매우 적합)
    - 수식적으로 분모에 과거의 모든 Gradient 제곱의 합($G_t$)을 누적하여 루트를 씌워 나눠줍니다.
- 💡 [치명적 단점]: 제곱의 합을 계속 더하기만 하므로 분모가 무한히 커집니다. 학습이 진행될수록 학습률이 0으로 수렴(Infinitesimally small)해버려, 결국 학습이 완전히 멈추는 현상이 발생합니다.

### 2) Adadelta & RMSprop
- 개념: Adagrad의 '학습률이 0으로 소멸하는 문제'를 해결하기 위해 등장했습니다.
- 원리 (RMSprop 기준): 과거의 모든 Gradient 제곱을 더하는 대신, 지수 이동 평균(Exponentially decaying average)을 사용합니다.
    - 즉, 아주 오래된 과거의 Gradient 정보는 점차 잊어버리고, 최근의 Gradient 정보만 주로 반영하여 분모가 무한히 커지는 것을 막습니다.
        $$
        E[\nabla w^2]_t = 0.9 E[\nabla w^2]_{t-1} + 0.1 \nabla w_t^2
        $$
- Adadelta의 특징: RMSprop과 유사하나, 파라미터 업데이트 시 '단위(Units)'까지 맞춰주기 위해 파라미터 업데이트의 RMS까지 계산식에 포함시킨 더 복잡한 버전입니다.

## 18. 👑 최적화 알고리즘의 끝판왕: Adam (Adaptive Moment Estimation)
- 개념: 방향성을 잡아주는 Momentum의 장점과, 파라미터별로 학습률을 조절해 주는 RMSprop의 장점을 하나로 합친 알고리즘입니다.
- 원리: 과거 Gradient의 지수 이동 평균(Momentum 역할, $m_t$)과 과거 Gradient 제곱의 지수 이동 평균(RMSprop 역할, $v_t$)을 모두 계산하여 업데이트에 사용합니다.
- 💡 Bias-correction (편향 보정): $m_t$와 $v_t$를 처음에 0으로 초기화하기 때문에, 학습 초반에는 0에 가깝게 편향(Biased)되는 문제가 있습니다. 이를 방지하기 위해 $1-\beta^t$로 나눠주는 보정 작업을 거칩니다.

> 👨‍🏫 교수님의 실전 꿀팁 (농담 섞인 조언):
> "결국엔 다른 거 안 쓰고 아담(Adam) 계열을 쓰고 있을 겁니다. 여러분들은 고민할 것 없이 그냥 아담 계열을 쓰시면 됩니다."

## 📝 최종 요약 및 시험 대비 체크리스트
- SVM vs FNN: Kernel(사용자 정의) vs Data-driven Basis Function(학습)
- Activation Function: Step $\rightarrow$ Sigmoid/Tanh (미분 용이) $\rightarrow$ ReLU (Vanishing Gradient 완화)
- Data Batching: Batch (신뢰도 높음, Local Optima 취약) vs Stochastic (궤적 불안정, Local Optima 탈출 가능, Shuffling 필수) vs Mini-Batch (표준)
- Backpropagation: Chain Rule을 이용해 출력층 에러를 입력층 쪽으로 책임 전가 (이때 1보다 작은 미분값이 계속 곱해져 Vanishing Gradient 유발)
- Computational Graph: 수동 미분의 고통을 없애고 Auto-Differentiation을 가능하게 한 혁신.
- DNN 한계 극복: Pre-training (Auto-Encoder 활용), ReLU, Dropout (동조화 방지), Batch Normalization (Internal Covariate Shift 해결)
- Optimizers: Momentum (관성) / NAG (미리 가보고 교정) / Adagrad (개별 학습률, 0으로 소멸 단점) / RMSprop (소멸 방지) / Adam (Momentum + RMSprop)
