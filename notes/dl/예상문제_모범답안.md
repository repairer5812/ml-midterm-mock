## 문제 1 — RNN 파라미터 공유 (7주차)

### 🎯 기출 연관
2025 기출 2-(2) 직계: "RNN이 파라미터를 어떻게 공유하고, 그걸 안 하면 무슨 일이 일어나는지"

### 📝 문제
> RNN은 모든 타임스텝에서 동일한 가중치 행렬 $W$를 공유한다.
> - (1) 이 파라미터 공유가 이루어지는 메커니즘을 설명하시오.
> - (2) 만약 타임스텝마다 다른 $W$를 두면, 테스트 시 RNN의 "임의 길이(Arbitrary length) 처리"라는 핵심 장점이 어떻게 무너지는지 서술하시오.
> - (3) BPTT로 $W$의 Gradient를 구할 때 각 타임스텝 편미분의 합(Sum)이 되는 이유를 공유 구조와 연결해 논하시오.

### 🔑 핵심 키워드 (채점 포인트)
- `동일 함수 $f_W$`, `매 시점 반복 호출`, `State Vector`, `Unfold`
- `Arbitrary length`, `임의 길이 시퀀스 처리`
- `Chain Rule`, `편미분의 합`, `공유된 변수 미분`

### 💯 모범답안

**(1) 파라미터 공유 메커니즘**

RNN의 상태 전이는 $h_t = f_W(h_{t-1}, x_t)$ 라는 단일 함수로 표현된다. 여기서 핵심은 "The same function and the same set of parameters are used at every time step" 이라는 원칙이다. 즉, $t=1, 2, 3, \ldots$ 어느 시점이든 동일한 함수 $f_W$를 반복 호출하며, 함수 내부의 가중치 행렬 $W$(입력 변환 $W_{xh}$, 상태 전이 $W_{hh}$ 등)는 모든 시점이 같은 메모리 주소를 공유한다. 시간축으로 Unfold한 그림에서는 여러 블록처럼 보이지만, 실제로는 단 하나의 $W$를 $T$번 사용하는 것이다.

**(2) 공유하지 않을 때의 문제**

만약 시점마다 $W_1, W_2, \ldots, W_T$를 따로 두면 파라미터 수가 $T$배로 늘어나는 것 이상의 구조적 결함이 생긴다. 학습 시에는 시퀀스 길이 $T=50$에 대해 $W_1 \sim W_{50}$까지만 학습되는데, 테스트 시 $T=100$짜리 시퀀스가 들어오면 $W_{51} \sim W_{100}$이 존재하지 않아 모델 자체가 적용 불가능해진다. 즉 "임의 길이 시퀀스에 유연 대응"이라는 RNN의 본질적 장점이 붕괴한다. 반대로 $W$를 공유하면 시퀀스를 몇 번 반복하든 동일한 $W$를 재사용하므로 길이 제약이 사라진다.

**(3) Gradient가 편미분의 합이 되는 이유**

$W$가 모든 시점에서 공유되므로, 최종 Loss $L = \sum_t L_t$에 대한 $W$의 미분은 연쇄법칙(Chain Rule)에 의해

$$\frac{\partial L}{\partial W} = \sum_{t=1}^{T} \frac{\partial L_t}{\partial W}$$

가 된다. 직관적으로 "하나의 $W$가 시점 1에서도, 시점 2에서도… 시점 $T$에서도 쓰였으니, 각 시점에서 $W$가 기여한 Gradient를 모두 더해주어야 올바른 업데이트 방향이 나온다"는 것이다. 이는 "공유된 변수의 Gradient는 그 변수가 등장한 모든 지점의 편미분을 합산한다"는 Computational Graph의 일반 원칙과 동일하다.

### ⚠️ 자주 빼먹는 함정
- "파라미터를 공유한다"만 쓰고 왜 공유해야 하는지(Arbitrary length) 를 놓침
- Gradient 합산의 수학적 근거(Chain Rule + 공유 변수) 를 생략하고 "그냥 합친다"고만 적음

## 문제 2 — 가변 이미지 CNN+RNN 설계 (5+7주차)

### 🎯 기출 연관
2025 기출 3번 직계: "이미지 크기 가변 · CNN+RNN 설계 · 장단점"

### 📝 문제
> 입력 이미지 크기가 가변적이고 출력으로 이미지 설명 문장(가변 길이)을 생성하는 시스템을 설계하시오.
> - (1) CNN+RNN 조합 구조를 글로 설계 — 각 모듈 역할, 연결점(초기 State Vector) 포함.
> - (2) VGG의 마지막 Softmax·Hidden 레이어를 제거하고 그 이전 레이어 출력을 쓰는 이유.
> - (3) 이 구조의 장점 2가지·단점 2가지를 논하시오.

### 🔑 핵심 키워드
- `Image Captioning`, `1:N 구조`, `Static 입력 → Sequence 출력`
- `VGG 사전학습`, `Classification 특화 vs Description용 하이레벨 특징`
- `초기 State Vector 주입`, `Auto-regressive`, `<Start>/<End> 토큰`
- `Global Average Pooling` (가변 크기 이미지 처리)

### 💯 모범답안

**(1) 구조 설계**

입력이 가변 크기 이미지이고 출력이 가변 길이 문장이므로 1:N (One-to-Many) RNN 구조를 CNN과 결합한다.

- CNN 인코더 (이미지 → 고정 크기 임베딩): ImageNet으로 사전학습된 VGG를 사용하되, 이미지 크기가 가변이므로 마지막 Convolutional Feature Map 뒤에 Global Average Pooling을 붙여 항상 동일한 차원의 임베딩 벡터 $v$를 뽑는다.
- RNN 디코더 (임베딩 → 문장): 추출된 $v$를 RNN의 초기 State Vector $h_0$ 로 주입한다. 즉 "이 이미지가 담고 있는 의미"가 초기 메모리 컨디션이 된다. 이후 `<Start>` 토큰을 입력으로 주면 단어를 하나씩 생성하고, 생성된 단어를 다음 시점의 입력으로 되먹임하는 Auto-regressive 방식으로 문장을 만든다. `<End>` 토큰이 나오면 종료한다.

**(2) VGG 마지막 레이어를 버리는 이유**

VGG의 최종 Softmax와 그 직전 Fully-Connected Hidden 레이어는 1,000개 ImageNet 클래스로 분류(Classification) 하는 데만 특화되어 있다. 이 단계에서는 "개/고양이/버스"처럼 카테고리 라벨에 필요한 정보만 남고, 이미지를 풍부하게 묘사(Describe)하는 데 필요한 하이레벨 특징(색감, 구도, 객체 간 관계 등)은 이미 소실된다. 캡셔닝은 분류가 아니라 묘사이므로, Classification head를 버리고 그 직전 Convolutional/Feature 레이어의 출력(더 일반적이고 풍부한 표현)을 임베딩으로 사용해야 한다.

**(3) 장점 2가지 · 단점 2가지**

장점
1. 모달 결합: CNN의 공간 인식력과 RNN의 시퀀스 생성력이 각자 강점을 살려 End-to-End 학습 가능.
2. 가변 입출력 대응: Global Average Pooling으로 입력 이미지 크기 제약을 없애고, RNN의 Auto-regressive + `<End>` 토큰으로 출력 문장 길이 제약도 없앰.

단점
1. 오류 전파(Error Propagation): RNN은 $t-1$ 시점의 잘못된 생성을 $t$ 시점의 입력으로 받기 때문에, 초반 단어 하나가 틀리면 뒷문장 전체가 무너질 수 있음.
2. GPU 병렬화 제한: RNN 부분이 순차 연산이라 학습 속도가 느리고, 최근 Transformer 기반 캡셔너에 비해 효율이 떨어짐.

### ⚠️ 자주 빼먹는 함정
- "VGG 쓴다"만 적고 왜 마지막 레이어를 버리는지 이유를 생략
- `<End>` 토큰의 역할을 잊고 "RNN이 알아서 멈춘다"고만 적음

## 문제 3 — 시계열에 CNN 적용 (7주차)

### 🎯 기출 연관
교수님 직접 오픈 질문 (강의 10:39~11:07): "시계열에 CNN은 쓸 수 없겠느냐? 잘 생각해보십시오."
→ 답 안 주고 넘어간 시험 출제 예고 떡밥

### 📝 문제
> 시계열 데이터(음성·주가·센서 시그널)에 전통적으로는 RNN을 사용해왔으나, 최근 CNN을 적용하는 경우도 많다.
> - (1) 시계열에 CNN을 적용할 때의 구조적 특징을 설명.
> - (2) RNN 대비 장점 2가지 서술.
> - (3) RNN 대비 단점 2가지 서술.

### 🔑 핵심 키워드
- `1D Convolution`, `커널`, `Receptive Field`
- `병렬 연산(Parallelization)`, `순차 연산 불필요`
- `로컬 패턴(Local Pattern)`, `단기 특징`
- `Arbitrary length 처리 어려움`, `패딩 필요`
- `Long-term Dependency` 부족

### 💯 모범답안

**(1) 구조적 특징**

시계열 데이터에 CNN을 적용할 때는 이미지용 2D Convolution 대신 1D Convolution을 사용한다. 커널을 시간축(길이 방향)으로만 슬라이드시키며 특정 윈도우 사이즈 내의 시간적 패턴을 추출한다. 예를 들어 커널 크기 5면 연속된 5개 타임스텝의 값을 한 번에 보고 하나의 Feature 값을 출력한다. 이를 여러 채널로 확장하면 다양한 시간 패턴을 동시에 검출할 수 있다.

**(2) RNN 대비 장점 2가지**

1. 병렬 연산(Parallelization) 가능: RNN은 $t$ 시점 연산이 $t-1$ 시점 State에 의존해 순차적으로만 계산해야 하므로 GPU 병렬 처리 이점을 살리기 어렵다. 반면 CNN은 모든 시점의 Convolution을 동시에 수행할 수 있어 학습·추론 속도가 매우 빠르다.
2. 로컬 패턴 추출 강력: CNN 커널은 특정 윈도우 내의 단기 패턴에 특화되어 있어, 음성의 특정 음소 파형·주가의 단기 급등락·센서의 스파이크 같은 짧은 시간 단위 특징을 매우 잘 포착한다.

**(3) RNN 대비 단점 2가지**

1. 임의 길이(Arbitrary length) 처리 제한: RNN은 시퀀스가 아무리 길어도 동일 $W$로 처리 가능하지만, CNN은 커널·입력 크기가 기본적으로 고정되어 있어 들쭉날쭉한 길이를 다루려면 Zero-padding·Global Pooling·마스킹 등 추가 장치가 필요하다.
2. 장기 의존성(Long-term Dependency) 파악 부족: RNN은 State Vector라는 "압축된 메모리"로 아주 먼 과거 정보까지 끌고 가지만, CNN은 지정된 Receptive Field 바깥의 정보는 보지 못한다. 문장 맨 앞 단어가 맨 뒤 단어 의미를 결정하는 것 같은 긴 문맥을 파악하는 데 불리하다. (이 한계를 완화하려고 Dilated Convolution, Attention 등이 추가 도입됨.)

### ⚠️ 자주 빼먹는 함정
- "CNN도 쓸 수 있다"만 쓰고 1D Convolution임을 명시하지 않음
- 장점 `병렬 연산`과 단점 `Long-term Dependency`를 RNN의 반대 특성으로 명확히 대비하지 않음

## 문제 4 — 전통 ML vs DL (2주차)

### 🎯 기출 연관
2025 기출 1번 직계: "전통적인 기계학습과 딥러닝의 차이점"

### 📝 문제
> 전통적인 기계학습과 딥러닝의 차이점을 Feature Extraction 담당 주체(사람 vs 기계) 관점에서 설명하고, 딥러닝이 오히려 전통 ML보다 불리한 경우가 있다면 구체적 상황과 근거로 서술하시오.

### 🔑 핵심 키워드
- `수작업 Feature Engineering` (전통 ML) vs `End-to-End 학습` (DL)
- `도메인 지식 의존성`
- `소규모 데이터셋`, `해석 가능성(Interpretability)`, `Tabular data`
- `데이터 효율성`, `오버피팅 위험`

### 💯 모범답안

차이점: Feature Extraction 주체

전통 기계학습에서는 사람(도메인 전문가) 이 원시 데이터에서 의미 있는 특징을 수작업으로 설계한다. 예를 들어 컴퓨터 비전에서는 SIFT·HOG·Gabor 필터 같은 핸드크래프트 특징을 뽑은 뒤 SVM이나 랜덤 포레스트에 넣었다. 이 과정이 ML 프로젝트에서 가장 많은 노력이 들어가는 부분이었다.

반면 딥러닝은 Feature Extraction을 기계 스스로 학습한다. 입력 → 여러 Hidden Layer → 출력의 구조에서 Hidden Layer들이 계층적으로 저수준(edge) → 중수준(motif) → 고수준(object) 특징을 자동으로 찾아낸다. 사람이 특징을 직접 설계할 필요가 없어져 End-to-End 학습이 가능해지고, 도메인 지식 없이도 충분한 데이터만 있으면 높은 성능을 낼 수 있다.

딥러닝이 불리한 경우

1. 소규모 데이터셋: 딥러닝은 파라미터 수가 매우 많아 수천~수만 개 이상의 데이터가 있어야 일반화된다. 데이터가 수백 개밖에 없는 의료 희귀 질환 분류나 산업 소량 결함 검출 문제에서는 오버피팅이 심각하다. 이때는 핸드크래프트 특징 + SVM/랜덤 포레스트가 안정적으로 더 좋은 성능을 낸다.
2. 해석 가능성이 중요한 경우: 금융·의료처럼 "왜 이 결정을 내렸는지" 설명이 필수인 영역에서는 딥러닝의 블랙박스 특성이 문제가 된다. 결정 트리·로지스틱 회귀 같은 전통 ML은 각 feature의 기여도를 수식으로 직접 볼 수 있어 규제·감사·신뢰성 측면에서 유리하다.
3. (추가) 정형 데이터(Tabular data)에서는 여전히 XGBoost 같은 Gradient Boosting이 딥러닝을 능가하는 경우가 많다. 정형 데이터는 계층적 패턴이 약해 DL의 강점이 덜 발휘되기 때문이다.

### ⚠️ 자주 빼먹는 함정
- 단순히 "딥러닝이 더 깊다/레이어가 많다"만 적음 → Feature Extraction 주체 차이라는 본질을 놓침
- "딥러닝 불리한 경우" 문항을 생략. 교수님은 양면 비교를 원함

## 문제 5 — 오토인코더 Pre-training (3주차)

### 🎯 기출 연관
2025 기출 2-(1) 직계: "오토인코더의 활용방안"

### 📝 문제
> 오토인코더(Auto-Encoder)의 구조와 학습 방식을 설명하고, 이것이 2006년 딥러닝 부흥기에 Vanishing Gradient를 우회하는 Greedy Layer-wise Pre-training 기법에 어떻게 활용되었는지 서술하시오.

### 🔑 핵심 키워드
- `Encoder`, `Decoder`, `Bottleneck` 구조
- `Target = 입력 자기 자신` (Self-supervised)
- `Vanishing Gradient`, `Greedy Layer-wise Pre-training`
- `제프리 힌튼`, `Fine-tuning`, `의미 있는 초기값`
- `Denoising Auto-Encoder` (추가 포인트)

### 💯 모범답안

오토인코더 구조 및 학습

오토인코더는 Encoder(입력 → 저차원 Hidden) + Decoder(Hidden → 입력 차원 복원) 로 이루어진 2단계 신경망이다. 특이한 점은 출력층의 Target이 입력값 자기 자신이라는 것이다. 즉 네트워크는 입력 $x$를 한 번 압축했다가 다시 $x$로 복원하는 능력을 학습한다. 이 과정에서 Hidden Layer는 입력의 본질적 특징을 담은 압축 표현(Compressed Representation) 을 스스로 찾게 된다. 라벨이 필요 없으므로 Self-supervised 학습 방식이다.

Greedy Layer-wise Pre-training의 원리

2006년 이전에는 레이어를 깊게 쌓으면 Vanishing Gradient(Sigmoid·Tanh 미분값이 1보다 작아 역전파 중 0으로 수렴) 때문에 입력층 근처 가중치가 학습되지 않아 깊은 망 학습이 사실상 불가능했다. 제프리 힌튼은 "한 번에 전체를 학습시키지 말고, 레이어별로 잘라서 먼저 학습시키자"는 아이디어를 제안했다.

절차:
1. 원본 데이터로 첫 오토인코더를 학습 → Encoder 부분만 남기고 Decoder는 버림 → 첫 Hidden Layer 가중치 $W_1$ 고정.
2. 고정된 $W_1$의 출력을 입력으로 삼아 두 번째 오토인코더 학습 → $W_2$ 확보.
3. 원하는 깊이까지 반복하며 $W_3, W_4, \ldots$ 쌓음.
4. 마지막에 Classification 레이어를 붙여 Fine-tuning (전체 네트워크를 한 번에 지도학습으로 미세 조정).

이 방식은 각 레이어가 이미 의미 있는 초기값을 가진 상태로 시작하므로 Vanishing Gradient의 영향이 훨씬 작고, 처음으로 깊은 네트워크가 실제로 학습되어 딥러닝 부흥의 기폭제가 되었다. 추후 ReLU와 BatchNorm이 Vanishing Gradient를 더 근본적으로 해결하면서 이 Pre-training 기법은 생성 모델·표현 학습 쪽으로 역할이 이동했지만, 딥러닝 재도약의 결정적 전환점이었다는 역사적 의의가 크다.

추가 활용: Denoising Auto-Encoder(노이즈 입력 → 원본 복원)로 더 Robust한 특징을 학습하거나, 현재의 Variational Auto-Encoder(VAE) 및 Diffusion 모델의 이론적 기반이 되었다.

### ⚠️ 자주 빼먹는 함정
- 오토인코더 구조만 설명하고 Pre-training 메커니즘을 연결하지 못함
- "깊은 망이 안 돼서 이렇게 했다"만 쓰고 Vanishing Gradient의 수학적 원인을 언급하지 않음

## 문제 6 — CIFAR-10 실험 분석 (6주차)

### 🎯 기출 연관
6주차 과제 힌트 직격: "어떤 시도를 했을 때 어떤 차이가 있었는지 그 이유를 설명"

### 📝 문제
> 본인이 CIFAR-10 CNN 분류기에 기법을 하나씩 누적하며 성능을 측정했다.
> ```
> Baseline CNN (ReLU + SGD)      : 56%
> + Leaky ReLU                    : 58%
> + Weight Decay 0.01             : 59%
> + He Initialization             : 61%
> + Batch Normalization           : 67%
> + Data Augmentation             : 72%
> ```
> - (1) Leaky ReLU가 +2%p 준 이유를 Dead ReLU 관점에서 설명.
> - (2) He Init이 Weight Decay·Dropout 뒤에 와서도 +2%p를 준 이유를 ReLU와의 관계로 논하시오.
> - (3) Batch Norm이 +6%p로 가장 큰 점프를 만든 이유를 Internal Covariate Shift 관점에서 서술.

### 🔑 핵심 키워드
- `Dead ReLU`, `기울기 0`, `음수 기울기 0.01`
- `He Init 분산 스케일`, `ReLU 특화 초기화`
- `Internal Covariate Shift`, `레이어별 입력 분포 안정화`
- `Scale/Shift 파라미터`

### 💯 모범답안

**(1) Leaky ReLU가 +2%p 기여**

기존 ReLU는 음수 입력에 대해 출력을 0으로 강제하는데, 이때 Gradient도 0이 되어 해당 뉴런은 역전파로 업데이트되지 않는 Dead ReLU 상태에 빠진다. 한번 죽은 뉴런은 계속 0만 출력해 학습에서 영구 제외되어 모델의 유효 용량(effective capacity)이 줄어든다. Leaky ReLU는 음수 영역에 작은 기울기(예: 0.01) 를 부여해 Gradient가 완전히 0이 되지 않게 한다. 죽어버릴 뻔한 뉴런들이 살아나 모델 전체의 표현력이 회복되며 성능이 향상된다.

**(2) He Init이 이미 개선된 상태에서 +2%p 기여**

Weight Decay와 Dropout은 정규화(Overfitting 억제) 기법이고, He Initialization은 학습 시작점(초기 가중치 분포) 을 설정하는 기법이라 해결하는 문제 층위가 다르다. ReLU는 음수 입력의 Activation을 0으로 죽이므로, 평균적으로 입력 분산의 절반만 다음 레이어로 전달된다. 일반적인 Xavier Init은 Tanh 기준으로 설계되어 이 특성을 반영하지 못하고, 깊은 망에서는 레이어가 지날수록 Activation 분산이 0으로 수렴하거나 폭발한다. He Init은 $W \sim \mathcal{N}(0, 2/n_{in})$ 로 ReLU의 "절반 통과" 특성을 명시적으로 보정해, 레이어 깊이에 관계없이 Activation과 Gradient의 분산이 안정적으로 유지된다. 그 결과 학습 초반부터 Gradient가 건강하게 흐르며 오버피팅이 아닌 근본적인 학습 효율 개선이 일어나 추가 +2%p가 가능하다.

**(3) BatchNorm이 +6%p로 가장 큰 점프**

네트워크가 학습되면서 각 레이어의 가중치가 바뀌면, 그 레이어의 출력 분포(즉 다음 레이어의 입력 분포)가 끊임없이 변한다. 이를 Internal Covariate Shift라 한다. 다음 레이어는 분포가 계속 바뀌는 입력을 쫓아가느라 학습이 불안정해지고 Learning Rate도 보수적으로 잡을 수밖에 없다.

BatchNorm은 각 미니배치 단위로 중간 Activation을 평균 0, 분산 1로 정규화해 분포를 강제로 고정시킨다. 이로 인해:
1. 레이어 간 입력 분포가 안정되어 학습이 빠르게 수렴.
2. Gradient 흐름이 개선되어 더 깊은 망도 학습 가능.
3. 큰 Learning Rate 사용이 가능해져 실질적인 최적화 성능 상승.
4. 약한 정규화(Regularization) 효과 부수적으로 얻음.

강제 정규화로 표현력이 떨어질 수 있으므로 학습 가능한 Scale($\gamma$)·Shift($\beta$) 파라미터를 두어 필요 시 복원한다. 이 네 효과가 동시에 누적되기 때문에 다른 단일 기법보다 압도적으로 큰 +6%p 점프가 발생한다.

### ⚠️ 자주 빼먹는 함정
- Leaky ReLU 답에서 기울기 0 → 업데이트 안 됨 논리를 생략
- He Init 답에서 "ReLU를 위한 초기화"만 쓰고 분산 스케일 수식적 근거를 빠뜨림
- BatchNorm 답에서 Scale·Shift 파라미터 언급 생략

## 문제 7 — RNN GPU 병렬화 한계 (7주차)

### 🎯 기출 연관
7주차 RNN 치명적 단점 섹션, 교수님이 "Transformer 등장 이유" 로 강조.

### 📝 문제
> RNN이 최근 Transformer에 밀린 가장 핵심적인 이유는 GPU 병렬 처리의 한계다.
> - (1) 왜 $t$ 시점 연산에 $t-1$ 결과가 구조적으로 필요한지 State Vector 관점에서 설명.
> - (2) Transformer가 이 문제를 어떻게 해결했는지 (교수님 언급 수준에서).
> - (3) 최근 주목받는 Mamba 모델의 특징을 한 문장으로.

### 🔑 핵심 키워드
- `State Vector`, `순차 의존성(Sequential Dependency)`
- `GPU 병렬화`, `Wall-clock 시간`
- `Self-Attention`, `동시 처리`, `토큰 간 거리 독립`
- `Mamba`, `State Space Model`, `병렬 가능한 순환`

### 💯 모범답안

**(1) RNN이 순차 연산에 묶이는 이유**

RNN의 상태 전이 식 $h_t = f_W(h_{t-1}, x_t)$ 에서 $h_t$를 계산하려면 반드시 $h_{t-1}$이 먼저 계산되어 있어야 한다. $h_{t-1}$은 다시 $h_{t-2}$에 의존하므로, 결국 시퀀스 맨 앞부터 끝까지 한 시점씩 순서대로 계산할 수밖에 없다. State Vector는 "과거의 모든 히스토리가 압축된 메모리"이기 때문에 이 순차 의존성은 RNN의 본질이며 우회할 방법이 없다. GPU는 수천 개 연산을 동시에 처리할 수 있어도 RNN은 그 병렬성을 시간축으로 활용할 수 없고, 시퀀스 길이 $T$에 비례해 Wall-clock 시간이 선형으로 증가한다.

**(2) Transformer의 해결**

Transformer는 순환(Loop)을 아예 제거하고 Self-Attention이라는 새로운 메커니즘을 쓴다. Self-Attention은 시퀀스의 모든 토큰 간 관계를 행렬 곱 한 번으로 동시에 계산한다. 즉, $t$ 시점이 $t-1$ 시점에 의존하지 않고, 모든 시점이 서로를 병렬로 참조한다. 덕분에 GPU 병렬성이 완전히 활용되어 RNN 대비 학습·추론 속도가 압도적으로 빠르고, 장기 의존성도 거리와 무관하게 바로 연결할 수 있다.

**(3) Mamba의 특징 (한 문장)**

Mamba는 RNN처럼 순차적인 State Space Model의 구조를 유지하면서도 하드웨어 병렬 처리가 가능하도록 재설계해, Transformer의 병렬성과 RNN의 효율적 메모리를 모두 노린 최신 모델이다.

### ⚠️ 자주 빼먹는 함정
- "RNN은 순차적이다"만 쓰고 State Vector 의존 고리를 명시하지 않음
- Transformer 해결책을 "Attention을 쓴다"로 단순화하고 "모든 토큰을 동시에 본다" 는 병렬성 핵심을 놓침
