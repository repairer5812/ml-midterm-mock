## 문제 1 (W2) — 전통 ML vs DL · Polynomial Curve Fitting과 Overfitting/Regularization

### 문제

> 다항식 곡선 피팅(Polynomial Curve Fitting, Bishop) 예제를 사용하여 다음을 서술하시오.
>
> - (1) 전통적 기계학습과 딥러닝의 핵심 차이점을 Feature Extraction의 주체 관점에서 설명하시오.
> - (2) 차수 $M$이 0, 1, 3, 9일 때 각각 어떤 일이 일어나는지 곡선 형태와 가중치 $W$의 크기 변화를 함께 설명하고, $M=9$일 때 발생하는 Overfitting의 증상을 서술하시오.
> - (3) Regularization(정규화) 항을 손실 함수에 추가했을 때 $\lambda$ 값에 따라 어떤 일이 발생하는지, $\ln \lambda = -\infty$, $-18$, $0$ 세 경우를 비교하여 설명하시오.

### 핵심 키워드

- 전통 ML (Machine Learning): `수작업 Feature Engineering` · `도메인 전문가` · `SIFT/HOG/Gabor` · `SVM/RF`
- 딥러닝: `End-to-End` · `Hidden Layer 계층적 학습` · `Edge → Motif → Object`
- Polynomial: `$y(x,W) = \sum_{j=0}^{M} w_j x^j$` · `$W$ 관점 선형` · `Global Optimum`
- Overfitting: `훈련 오차 0` · `$W$ 계수 폭증` · `테스트 오차 발산`
- Regularization: `$E(W) = \frac{1}{2}\sum (y - t)^2 + \frac{\lambda}{2}\|W\|^2$` · `$\lambda$ 너무 크면 Underfitting`

### 모범답안

(1) Feature Extraction 주체의 차이

전통 기계학습에서는 사람(도메인 전문가)이 원시 데이터에서 의미 있는 특징을 수작업으로 설계한다. 컴퓨터 비전이라면 SIFT·HOG·Gabor 필터로 핸드크래프트 특징을 뽑은 뒤 SVM이나 랜덤 포레스트에 입력했다. 강의에서 교수님은 "MNIST 같은 28×28 이미지에 7×7 마스크를 직접 씌워 유의미한 부분을 뽑아내는 작업이 ML 프로젝트의 가장 큰 노력 부분이었다"고 강조했다. 반면 딥러닝은 입력 → Hidden Layer → 출력 구조에서 Hidden Layer가 스스로 저수준(edge) → 중수준(motif) → 고수준(object) 특징을 계층적으로 학습한다. 즉 Feature Extraction과 Classifier가 한 네트워크 안에서 End-to-End로 학습되며, 사람이 도메인 지식을 동원해 특징을 설계할 필요가 사라진 것이다.

(2) 다항식 차수 $M$과 Overfitting

가설은 $y(x, W) = w_0 + w_1 x + w_2 x^2 + \cdots + w_M x^M$ 이다. 이 식은 입력 $x$ 관점에서는 비선형이지만 파라미터 $W$ 관점에서는 1차식이라 미분 시 유일한 글로벌 옵티멈이 존재한다. 차수 $M$이 커질수록 곡선이 구부러질 수 있는 횟수가 늘어나 모델의 표현력(Capacity)이 증가한다.

- $M=0$: 상수만 학습. 직선이 아닌 평행선. 데이터의 사인 형태를 전혀 잡지 못함 (Underfitting).
- $M=1$: 1차 직선. 여전히 사인 곡률을 못 따라감.
- $M=3$: 데이터 10개에 적당한 곡선이 만들어져 사인을 가장 잘 근사. 일반화 성능이 가장 좋음.
- $M=9$: 데이터 점 10개를 전부 통과하므로 훈련 오차가 0으로 떨어진다. 그러나 점 사이를 심하게 요동치며, 이때 가중치 $w_j$가 $|w_j|=10^{4}$ 단위로 비정상적으로 커진다 (Bishop 책 표 1.1). 새 데이터가 들어오면 예측이 완전히 무너지며 테스트 오차가 폭증한다. 이것이 Overfitting의 전형적 증상이다.

(3) Regularization과 $\lambda$의 효과

Overfitting의 증상이 "$W$ 계수가 폭증"하는 것이므로, 손실 함수에 $\|W\|^2$ 페널티를 직접 추가해 큰 $W$를 억제한다.

$$\tilde{E}(W) = \frac{1}{2} \sum_{n=1}^{N} \big(y(x_n, W) - t_n\big)^2 + \frac{\lambda}{2} \|W\|^2$$

- $\ln \lambda = -\infty$ ($\lambda = 0$): 페널티 없음. $M=9$일 때 정통 Overfitting 곡선이 그대로 나옴.
- $\ln \lambda = -18$ ($\lambda \approx 1.5 \times 10^{-8}$): 적당한 페널티. 곡선이 부드러워지고 사인 곡선에 가깝게 복원됨. 가중치 크기도 정상 범위로 들어옴.
- $\ln \lambda = 0$ ($\lambda = 1$): 페널티가 너무 강해 $W \to 0$으로 짓눌림. 곡선이 다시 평평해져 Underfitting이 발생.

즉 $\lambda$는 모델이 데이터를 외우는 정도와 가중치를 작게 유지하는 정도 사이의 트레이드오프를 조절하는 하이퍼파라미터다. 모델 복잡도($M$)를 직접 줄이는 대신 $\lambda$로 같은 효과를 부드럽게 얻을 수 있다는 것이 Regularization의 핵심 통찰이다.

### 자주 빼먹는 함정

- (1)에서 "딥러닝이 더 깊다"만 쓰고 Feature Extraction 주체 차이라는 본질을 놓침
- (2)에서 $W$ 계수가 폭증한다는 점을 빼먹고 "곡선이 요동친다"만 적음. 교수님은 가중치 절댓값 변화를 핵심 출제 포인트로 강조함
- (3)에서 $\lambda$가 너무 클 때 Underfitting이 된다는 양면성을 빠뜨림 — 모델 복잡도와의 트레이드오프 서술 필수
- $W$ 관점에서 선형이라 Global Optimum이 유일하다는 사실이 왜 중요한지 설명하지 않음

## 문제 2 (W3) — Backpropagation · Vanishing Gradient · DNN 부흥 전환점

### 문제

> Multi-Layer Perceptron(MLP)의 학습은 Backpropagation 알고리즘에 기반한다.
>
> - (1) Backpropagation의 기본 아이디어를 Hidden Layer가 "정답을 모른다"는 점과 연결해 설명하고, Chain Rule이 어떻게 활용되는지 수식적으로 서술하시오.
> - (2) Sigmoid·Tanh를 활성화 함수로 쓸 때 Vanishing Gradient 문제가 발생하는 수학적 원인을 미분값과 연쇄 곱셈 관점에서 설명하시오.
> - (3) 2006년 이후 딥러닝 부흥기에 이 문제를 해결한 두 가지 전환점(Auto-Encoder 기반 Pre-training, ReLU (Rectified Linear Unit) 도입)을 각각 어떻게 작동하는지 서술하시오.

### 핵심 키워드

- `Hidden Layer = 정답 없음` · `책임 전가(Credit Assignment)`
- `Chain Rule` · `$\delta_j = f'(a_j) \sum_k w_{kj} \delta_k$` · `약분 효과`
- `Sigmoid 미분 최댓값 0.25` · `Tanh 미분 최댓값 1` · `미분값 < 1 반복 곱`
- `Greedy Layer-wise Pre-training` · `힌튼 2006` · `의미 있는 초기값`
- `ReLU` · `양의 영역 미분 = 1` · `Dead ReLU`

### 모범답안

(1) Backpropagation의 기본 아이디어와 Chain Rule

MLP에서 출력층(Output Layer)은 정답 $t$가 있어 손실 $L = \frac{1}{2}(y-t)^2$를 직접 계산할 수 있지만, 중간 Hidden Layer에는 정답이 없다. 그래서 "히든(Hidden, 숨겨진) 레이어"라 부른다. Backpropagation은 이 문제를 출력에서 발생한 에러를 거꾸로 거슬러 올라가며 각 가중치에 책임을 분배(Credit Assignment)하는 방식으로 해결한다. 직관적으로는 "에러 10이 났을 때, 너희들이 연결된 가중치만큼 책임이 있으니 그 비율로 나눠 가져라"는 식이다.

수식으로는 손실 $L$을 가중치 $w_{ji}$로 미분할 때 Chain Rule을 적용해

$$\frac{\partial L}{\partial w_{ji}} = \frac{\partial L}{\partial a_j} \cdot \frac{\partial a_j}{\partial w_{ji}} = \delta_j \cdot z_i$$

가 되며, $\delta_j$는 다음 레이어의 $\delta_k$들로부터 재귀적으로 정의된다.

$$\delta_j = f'(a_j) \sum_k w_{kj} \delta_k$$

교수님은 이를 "분자·분모에 같은 항이 약분되며 끝(에러)에서 시작점(가중치)까지 미분값이 쫙 곱해져 전달된다"고 표현했다. 핵심은 각 레이어를 거칠 때마다 활성화 함수의 미분 $f'(a)$가 곱해진다는 점이며, 바로 이것이 Vanishing Gradient의 씨앗이다.

(2) Vanishing Gradient의 수학적 원인

Sigmoid의 미분은 $\sigma'(x) = \sigma(x)(1-\sigma(x))$ 로 최댓값이 $x=0$에서 $0.25$이며, 입력이 조금만 커도 0에 가까워진다. Tanh도 미분 최댓값은 1이지만 양 끝에서 빠르게 0으로 수렴한다. 깊이 $L$의 망에서는 입력층 근처 가중치의 그래디언트가

$$\frac{\partial L}{\partial w^{(1)}} \propto \prod_{l=1}^{L} f'(a^{(l)}) \cdot \prod w^{(l)}$$

형태로, 1보다 작은 미분값이 $L$번 반복 곱해진다. 예를 들어 Sigmoid에서 $0.25^{10} \approx 10^{-6}$ 이므로 입력층 근처는 사실상 학습 신호를 받지 못한다. 이 때문에 2010년대 이전에는 깊은 망이 실용적으로 학습 불가능했고, 딥러닝의 3대 난제 중 하나로 꼽혔다.

(3) 부흥기의 두 전환점

첫째, Auto-Encoder 기반 Greedy Layer-wise Pre-training (Hinton, 2006). Auto-Encoder는 입력을 압축했다가 다시 복원하는 Encoder-Decoder 구조로, Target이 입력 자기 자신이라 라벨이 필요 없는 Self-supervised 학습이다. 힌튼은 깊은 망 전체를 한꺼번에 학습시키지 말고 "한 레이어씩 잘라서 Auto-Encoder로 사전학습한 뒤 Encoder만 남기고 다음 레이어를 또 사전학습하는 식으로 쌓아 올린다." 마지막에 Classification 레이어를 붙여 전체를 Fine-tuning하면, 각 레이어가 이미 의미 있는 초기값을 가진 상태에서 시작하므로 Vanishing Gradient의 영향이 크게 줄어든다. 이것이 깊은 망 학습이 처음으로 안정적으로 성공한 사건이며, 2006년 딥러닝 부흥의 기폭제다.

둘째, ReLU 도입 (AlexNet, 2012). $\text{ReLU}(x) = \max(0, x)$의 미분은 양의 영역에서 정확히 1이라 아무리 곱해도 1로 유지된다. Vanishing Gradient를 근본적으로 해결한 것이다. AlexNet은 ImageNet에서 ReLU + Dropout + 두 GPU 병렬화로 압도적 성적을 내며 딥러닝 시대를 열었다. 다만 음수 입력에서 출력과 미분이 모두 0이 되는 Dead ReLU 부작용이 있어, 후속 연구에서 Leaky ReLU 등으로 보완되었다. 이후 BatchNorm·Residual Connection까지 더해지며 Pre-training 없이도 깊은 망을 학습하게 되었고, Pre-training 기법 자체는 표현 학습·생성 모델 쪽으로 역할이 이동했다.

### 자주 빼먹는 함정

- "Hidden Layer가 정답 없음 → Backprop 필요"라는 모티베이션을 생략
- Sigmoid 미분 최댓값이 0.25라는 구체적 수치를 놓침. 1보다 작다는 사실을 그냥 던지면 약분 안 됨
- ReLU만 답하고 2006년 Auto-Encoder Pre-training의 역사적 의의(부흥 전환점) 누락
- "ReLU가 모든 걸 해결"이라 단정하고 Dead ReLU 부작용 미언급

## 문제 3 (W4) — PyTorch nn.Module 모델 정의 · 학습 5단계 루프 · Tensor 변환의 필요성

### 문제

> PyTorch에서 신경망을 정의하고 학습시키는 표준 절차에 대해 다음에 답하시오.
>
> - (1) `nn.Module`을 상속하여 모델 클래스를 정의할 때 반드시 구현해야 하는 두 메서드(`__init__`, `forward`)의 역할을 설명하고 간단한 MLP (Multi-Layer Perceptron) 예시 코드를 제시하시오.
> - (2) 학습 루프 한 step에서 일어나는 5단계(zero_grad → forward → loss → backward → step)를 순서대로 설명하고, `optimizer.zero_grad()`를 빠뜨리면 어떤 문제가 생기는지 서술하시오.
> - (3) Pandas로 읽은 정형 데이터(DataFrame)를 모델에 넣기 전에 반드시 Tensor로 변환해야 하는 이유를 자동 미분과 GPU 가속 관점에서 설명하시오.

### 핵심 키워드

- `nn.Module 상속` · `super().__init__()` · `forward 자동 호출`
- `zero_grad → forward → loss → backward → step`
- `Gradient 누적 버그` · `이전 step gradient가 더해짐`
- `자동 미분(Autograd)` · `GPU 가속(.to('cuda'))` · `requires_grad`

### 모범답안

(1) `nn.Module` 기반 모델 클래스 정의

PyTorch는 모든 신경망 모델을 `torch.nn.Module`을 상속한 클래스로 정의한다. 두 메서드가 필수다.

- `__init__(self, ...)`: 학습 가능한 레이어들을 멤버 변수로 등록한다. 가장 먼저 `super().__init__()`을 호출해야 PyTorch 내부의 파라미터 자동 등록 기능이 작동한다.
- `forward(self, x)`: 입력 $x$에서 출력 $y$로 가는 순전파 계산을 정의한다. 학습/추론 시 `model(x)`로 호출하면 PyTorch가 내부적으로 `__call__`을 통해 자동으로 `forward`를 실행한다.

```python
import torch.nn as nn
import torch.nn.functional as F

class MLP(nn.Module):
    def __init__(self, in_dim=784, hidden=256, out_dim=10):
        super().__init__()
        self.fc1 = nn.Linear(in_dim, hidden)
        self.fc2 = nn.Linear(hidden, out_dim)

    def forward(self, x):
        h = F.relu(self.fc1(x))
        return self.fc2(h)
```

`__init__`에 등록된 `nn.Linear`는 자동으로 가중치 $W$와 편향 $b$를 생성하고 `requires_grad=True`로 설정해 자동 미분의 추적 대상이 된다.

(2) 학습 루프 5단계

PyTorch 표준 학습 루프 한 step은 다음 다섯 줄로 압축된다.

```python
for x, y in train_loader:
    optimizer.zero_grad()           # ① gradient 초기화
    y_hat = model(x)                # ② forward
    loss = criterion(y_hat, y)      # ③ loss 계산
    loss.backward()                 # ④ backward (자동 미분)
    optimizer.step()                # ⑤ 파라미터 업데이트
```

각 단계의 의미는 다음과 같다.

1. `zero_grad()`: 이전 step에서 계산된 gradient를 모두 0으로 초기화. PyTorch는 기본적으로 gradient를 누적(accumulate)하므로 이걸 빠뜨리면 이전 batch의 gradient가 계속 더해져 학습이 망가진다. RNN (Recurrent Neural Network)처럼 의도적으로 누적해야 할 때만 생략한다.
2. `forward`: 모델의 `forward(x)` 실행. Computational Graph가 자동으로 그려지며 모든 중간 텐서가 그래프 노드로 기록된다.
3. `loss = criterion(...)`: $L = \text{CrossEntropy}(\hat{y}, y)$ 같은 스칼라 손실 계산. 이 시점까지 그래프가 완성된다.
4. `loss.backward()`: 그래프를 거꾸로 traverse하며 Chain Rule로 모든 파라미터의 gradient를 자동 계산해 `param.grad`에 저장.
5. `optimizer.step()`: $W \leftarrow W - \eta \cdot \nabla_W L$ 식으로 파라미터를 실제로 업데이트.

`zero_grad()`를 빠뜨리는 것이 가장 흔한 입문자 버그다. 이전 batch의 gradient가 누적되면 사실상 batch size를 인위적으로 키운 듯한 효과가 나거나 발산해 버린다.

(3) Tensor 변환의 필요성

Pandas DataFrame이나 NumPy `ndarray`는 CPU 메모리상의 일반 배열로, 자동 미분(Autograd) 추적도 안 되고 GPU 연산도 불가능하다. PyTorch의 모든 학습 가능 연산은 Tensor 타입에서만 작동한다. Tensor의 두 가지 핵심 능력은 다음과 같다.

첫째, 자동 미분이다. Tensor는 `requires_grad=True` 속성을 가질 수 있고, 이 텐서로 수행된 모든 연산은 Computational Graph에 기록되어 `.backward()` 한 번으로 모든 파라미터 gradient를 자동 계산한다. NumPy로는 이게 불가능하므로 모든 미분을 수작업으로 짜야 한다.

둘째, GPU 가속이다. `tensor.to('cuda')` 한 줄로 텐서를 GPU 메모리에 올리면 행렬 곱·convolution 등이 수천 배 빨라진다. NumPy는 CPU 전용이라 이 가속을 활용할 수 없다.

따라서 학습 파이프라인은 항상 `pd.read_csv` → `df.values` (NumPy) → `torch.tensor(...)` (CPU Tensor) → `.to('cuda')` (GPU Tensor) 순으로 변환한 뒤 모델에 입력해야 한다. 그래야 자동 미분과 GPU 가속을 모두 활용할 수 있다.

### 자주 빼먹는 함정

- `super().__init__()` 호출을 빠뜨리면 파라미터 등록이 안 된다는 점 미언급
- 학습 루프 5단계에서 `zero_grad()`를 가장 마지막이나 중간에 두는 잘못된 순서로 적음 — 반드시 forward 직전(또는 직후 `loss` 계산 전)에 와야 함
- "Tensor가 GPU 가능하다"만 적고 자동 미분 추적이라는 더 본질적인 이유를 놓침

## 문제 4 (W5) — ResNet Skip Connection · Vanishing Gradient 해결 · DenseNet · Bottleneck

### 문제

> 깊이를 늘리는 것이 항상 성능 향상으로 이어지지는 않는다.
>
> - (1) ResNet (Residual Network) 이전에 발생한 Degradation Problem이 무엇인지 설명하고, 56-layer 망이 20-layer 망보다 오히려 에러율이 높아진 사례와 연결해 그 원인을 서술하시오.
> - (2) ResNet의 Skip Connection $H(x) = F(x) + x$가 어떻게 Vanishing Gradient를 해결하는지, 역전파 시 gradient가 어떻게 흐르는지 수식과 함께 설명하시오.
> - (3) DenseNet (Densely Connected Convolutional Network)은 ResNet의 덧셈을 Concatenation으로 바꾼 구조다. 두 방식의 정보 결합 방식 차이와 각각의 장단점을 서술하시오. 또한 ResNet에서 1×1 Conv를 활용한 Bottleneck Design이 왜 152-layer 같은 초깊이 망을 가능하게 했는지도 설명하시오.

### 핵심 키워드

- `Degradation` · `Overfitting과 다름` · `Vanishing/Exploding Gradient`
- `$H(x) = F(x) + x$` · `Identity Shortcut` · `잔차(Residual) 학습`
- `$\partial L / \partial x = \partial L / \partial H \cdot (1 + \partial F / \partial x)$` · `1이 더해져 죽지 않음`
- `DenseNet Concatenation` · `채널 누적` · `Feature Reuse`
- `1×1 Conv Bottleneck` · `채널 축소 → 3×3 → 채널 복원` · `152 layers`

### 모범답안

(1) Degradation Problem

원래 가설은 "망이 깊어지면 표현력이 늘어나 성능이 좋아진다"였다. 그러나 He et al.(2015)이 실험한 결과, 56-layer 평범한 CNN (Convolutional Neural Network)이 20-layer보다 훈련 오차마저 더 높았다. 이는 일반적인 Overfitting(훈련 오차는 0인데 테스트 오차가 높은 현상)과는 다른 별개의 문제로, "단지 망을 깊게 쌓는 것 자체가 최적화를 어렵게 만든다"는 Degradation Problem으로 명명되었다.

원인은 핵심적으로 Vanishing/Exploding Gradient다. 깊이가 깊어질수록 입력층 근처 가중치의 gradient가 활성화 함수 미분과 가중치 행렬의 반복 곱셈으로 0에 수렴(또는 폭발)해 학습 자체가 진행되지 않는다. Batch Normalization으로 일부 완화할 수 있었지만 근본적 해결은 되지 못했다.

(2) Skip Connection의 Vanishing Gradient 해결

ResNet은 입력 $x$를 몇 개의 레이어를 통과시킨 결과 $F(x)$에 원래 입력 $x$를 그대로 더해주는 구조를 도입한다.

$$H(x) = F(x) + x$$

학습 대상은 전체 매핑 $H(x)$가 아니라 잔차(Residual) $F(x) = H(x) - x$다. 만약 어떤 레이어 묶음이 항등 매핑이 최선이라면 $F(x) \to 0$으로 보내는 것이 단순 매핑을 통째로 학습하는 것보다 훨씬 쉽다.

역전파 시 그래디언트 흐름을 보면 그 위력이 더 분명하다.

$$\frac{\partial L}{\partial x} = \frac{\partial L}{\partial H} \cdot \frac{\partial H}{\partial x} = \frac{\partial L}{\partial H} \cdot \left(1 + \frac{\partial F}{\partial x}\right)$$

핵심은 괄호 안의 1이다. $\partial F / \partial x$가 활성화 함수 미분의 곱으로 0에 가까워지더라도 1이 더해져 있어 gradient가 결코 0으로 죽지 않는다. 즉 입력층까지 신호가 끊김 없이 전달된다(Uninterrupted Gradient Flow). 교수님은 "Skip Connection은 망 위로 뚫린 고속도로"라고 비유했다. 이 단순한 구조 덕분에 152-layer 같은 극단적 깊이도 학습 가능해졌고, 2015년 ILSVRC (ImageNet Large Scale Visual Recognition Challenge)를 휩쓸었다.

흥미롭게도 ResNet 1저자(Kaiming He)는 발표 당시 "왜 잘 되는지 모르겠는데 해보니 잘 됐다"고 답했고 1년 뒤에야 이론적으로 분석되었다는 일화도 있다.

(3) DenseNet과 Bottleneck Design

DenseNet은 같은 동기에서 출발했지만 결합 방식이 다르다. ResNet은 element-wise 덧셈 $F(x) + x$인 반면 DenseNet은 Concatenation $[F(x); x]$이다. 즉 채널 차원으로 이어 붙여 이전 레이어의 모든 feature가 그대로 보존된 채 다음 레이어에 전달된다.

| 구분 | ResNet | DenseNet |
|---|---|---|
| 결합 방식 | $F(x) + x$ (덧셈) | $[F(x); x]$ (채널 concat) |
| 정보 손실 | 덧셈으로 일부 섞임 | 원본 그대로 보존 |
| 파라미터 효율 | 큰 채널 유지 | Feature Reuse로 채널 작아도 OK |
| 메모리 | 작음 | 채널이 누적되어 메모리 큼 |

DenseNet의 장점은 Feature Reuse로 적은 파라미터로도 풍부한 표현을 만들 수 있고 gradient flow가 더 강하다는 것이다. 단점은 채널이 누적되어 GPU 메모리 사용량이 폭발적으로 늘어 실용성이 떨어진다는 점이다.

Bottleneck Design은 ResNet에서 1×1 Conv를 사용해 연산량을 극적으로 줄이는 기법이다. 기본 블록이 3×3 Conv 두 개인 데 비해, Bottleneck 블록은 (1×1 Conv로 채널 축소) → (3×3 Conv) → (1×1 Conv로 채널 복원) 3단 구조다. 예를 들어 256채널을 64채널로 줄여 3×3 Conv를 한 뒤 다시 256채널로 복원하면, 3×3 부분의 연산량이 $256 \times 256$이 아닌 $64 \times 64$ 스케일로 떨어져 약 16배 절약된다. 이 절약 덕분에 152-layer ResNet 같은 초깊이 망이 메모리·연산 한계 안에서 학습 가능해졌다. 1×1 Conv는 GoogLeNet에서 처음 본격 활용된 기법으로, 채널 차원에서 정보를 섞으면서 파라미터를 줄이는 효과를 함께 가진다.

### 자주 빼먹는 함정

- Degradation을 Overfitting으로 잘못 설명. 훈련 오차마저 높다는 점이 핵심
- Skip Connection이 vanishing을 해결하는 수식적 근거(1+∂F/∂x의 1)를 빼먹고 "그냥 정보가 잘 흐른다"고만 적음
- DenseNet 비교 시 메모리 폭증 단점을 누락
- Bottleneck을 단순히 "1×1로 줄였다"만 적고 152-layer를 가능하게 한 연산량 16배 절감 효과를 빼먹음

## 문제 5 (W6) — CNN이 DNN보다 이미지 분류에 우수한 이유 · Overfitting 방지 3기법 · He Init

### 문제

> CIFAR-10 이미지 분류 실습에서 다음을 답하시오.
>
> - (1) DNN (Deep Neural Network)(Fully-Connected MLP (Multi-Layer Perceptron))으로 CIFAR-10을 학습했을 때 약 51.65% 정확도에 그치는 이유를 공간 정보 보존과 가중치 공유 관점에서 설명하시오. CNN (Convolutional Neural Network)의 Conv-Pool-FC (Fully-Connected) 구조가 어떻게 이를 해결하는지도 함께 서술하시오.
> - (2) CIFAR-10에서 흔히 사용하는 Overfitting 방지 3기법(Early Stopping, Weight Decay, Dropout)의 원리와 적용 방법을 각각 설명하시오.
> - (3) ReLU (Rectified Linear Unit) 사용 시 He Initialization (Kaiming(He) Initialization)이 왜 Xavier (Xavier(Glorot) Initialization)보다 적합한지, 분산 스케일 관점에서 수식과 함께 서술하시오.

### 핵심 키워드

- DNN 한계: `Flatten → 공간 정보 손실` · `위치 변화에 취약` · `파라미터 폭증`
- CNN: `Local Receptive Field` · `Weight Sharing` · `Translation Invariance(Pooling)`
- Early Stopping: `Validation Loss 모니터링` · `Patience`
- Weight Decay: `$L_2$ 페널티` · `$L_{\text{total}} = L + \lambda \|W\|^2$` · `optimizer weight_decay 인자`
- Dropout: `$p$ 확률 비활성화` · `train/eval 모드 차이`
- He Init: `$W \sim \mathcal{N}(0, 2/n_{\text{in}})$` · `ReLU의 절반 통과 보정`

### 모범답안

(1) DNN의 한계와 CNN의 우위

CIFAR-10은 32×32×3 컬러 이미지다. DNN은 이걸 모델에 넣기 위해 `view(-1)`로 3,072차원 벡터로 Flatten한다. 이 순간 두 가지 핵심 정보가 파괴된다.

첫째, 공간적 인접성(Spatial Correlation)이 사라진다. 원래 옆에 있던 픽셀이 1D로 펴지면 멀리 떨어져 버려, MLP는 "이 두 픽셀이 인접하다"는 사실을 데이터로부터 다시 학습해야 한다. 둘째, 위치 변화에 매우 취약하다. 강아지 얼굴이 좌상단에 있는 사진들로만 학습했다면, 우하단으로 살짝 옮겨진 사진은 완전히 다른 입력으로 인식해 정확도가 폭락한다. 결과적으로 강의에서 본 DNN은 51.65% 정확도(랜덤 10%보단 낫지만 절반밖에 못 맞춤) 에 머문다. 더불어 첫 FC Layer만 해도 $3072 \times 512 \approx 157$만 파라미터로 너무 많아 Overfitting 위험도 크다.

CNN은 세 가지 메커니즘으로 이를 해결한다. Local Receptive Field(작은 윈도우만 보며 로컬 패턴 추출), Weight Sharing(같은 커널을 이미지 전체에 슬라이드해 위치와 무관하게 패턴 검출, 파라미터 수 급감), Subsampling/Pooling(공간 해상도를 줄여 계산량 감소 + Translation Invariance 획득)이다. 같은 강아지 얼굴이 어디 있어도 같은 커널로 검출되고, MaxPool 후엔 약간 옆으로 이동해도 결과가 거의 같다. CNN(Conv-Pool-FC) 구조로 가볍게 쌓기만 해도 약 56%로 오르고, 이후 He Init·BatchNorm·Augmentation까지 누적하면 70%대로 진입한다.

(2) Overfitting 방지 3기법

① Early Stopping (조기 종료). Train Loss는 계속 줄어드는데 Validation Loss가 더 이상 줄지 않거나 오히려 커지기 시작하는 시점에 학습을 강제 중단한다. 보통 Patience(예: 5 epoch 연속 개선 없음)를 두고 베스트 모델 가중치를 저장해 둔다. 강의 예에서는 100 epoch 예정이던 학습을 21 epoch에서 멈춰 불필요한 가중치 훼손을 막았다.

② Weight Decay (가중치 감쇠 / L2 Regularization). 손실 함수에 가중치 $L_2$ 노름을 페널티로 추가한다.

$$L_{\text{total}} = L + \lambda \|W\|^2$$

큰 $W$를 직접 억제해 모델이 특정 특징에 과도하게 의존하지 않게 한다. 구현은 PyTorch에서 옵티마이저 선언 시 `weight_decay=0.01` 인자만 추가하면 된다. 강의에서는 약 1%p 성능 향상을 보였다.

③ Dropout. 학습 중 각 뉴런을 확률 $p$(예: 0.5)로 무작위 비활성화한다. 매 mini-batch마다 다른 부분망으로 학습되어 앙상블 효과가 생기고, 특정 뉴런에 의존하는 co-adaptation을 끊는다. 평가(`model.eval()`) 시에는 모든 뉴런을 켜되 출력을 $1-p$ 비율로 스케일링해 분포를 맞춘다. `nn.Dropout(p=0.5)` 한 줄이면 된다.

세 기법은 서로 직교적이라 동시에 사용 가능하다.

(3) He Init과 ReLU의 분산 보정

가중치 초기화의 목표는 레이어를 통과할 때마다 활성화의 분산이 너무 커지거나(폭발) 작아지지 않게(소실) 유지하는 것이다. Xavier(Glorot) Init은 Tanh 같이 입력의 분산을 거의 그대로 통과시키는 활성화 함수를 가정해

$$W \sim \mathcal{N}\left(0, \frac{1}{n_{\text{in}}}\right)$$

로 초기화한다. 그런데 ReLU는 음수 입력의 출력을 0으로 죽이므로 평균적으로 분산의 절반만 다음 레이어로 전달된다. 따라서 Xavier로 초기화하면 깊은 ReLU 망에서 layer를 거칠수록 분산이 절반씩 줄어 결국 0에 수렴해 학습이 안 된다.

He Initialization은 이 절반 통과를 명시적으로 보정해

$$W \sim \mathcal{N}\left(0, \frac{2}{n_{\text{in}}}\right)$$

분산을 두 배로 키운다. ReLU의 0 죽임으로 절반이 빠지더라도 결과적으로 출력 분산이 입력 분산과 같게 유지되어, 어떤 깊이에서도 활성화와 gradient가 안정적으로 흐른다. 강의 실험에서 baseline에 He Init을 추가했을 때 +2%p 향상이 관찰되는 이유다. 즉 He Init은 정규화(Overfitting 방지)가 아니라 학습 시작점 자체를 ReLU에 맞게 설계해 Vanishing Gradient를 근본적으로 완화하는 기법이다.

### 자주 빼먹는 함정

- (1)에서 "DNN은 Flatten해서 공간 정보가 사라진다"만 적고, Weight Sharing으로 파라미터가 급감한다는 CNN의 두 번째 강점을 놓침
- (2)에서 Dropout을 "랜덤하게 끈다"만 적고 train/eval 모드 차이(eval 시 켜놓고 스케일링) 미언급
- He Init을 "ReLU에 맞춘 초기화"만 적고 분산 2/n 수식과 "절반 통과 보정"이라는 핵심을 빠뜨림

## 문제 6 (W7) — RNN 파라미터 공유 메커니즘 · 임의 길이 처리 · BPTT Gradient 합산

### 문제

> RNN (Recurrent Neural Network)은 모든 타임스텝에서 동일한 가중치 행렬 $W$를 공유한다.
>
> - (1) 이 파라미터 공유가 이루어지는 메커니즘을 $f_W$ 함수와 Unfold 그림 관점에서 설명하시오.
> - (2) 만약 타임스텝마다 다른 $W$를 두면 어떤 일이 일어나는지, 특히 RNN의 핵심 장점인 Arbitrary Length 처리 능력이 어떻게 무너지는지 서술하시오.
> - (3) BPTT(Backpropagation Through Time)로 $\partial L / \partial W$를 구할 때 결과가 각 타임스텝 편미분의 합(Sum)이 되는 이유를 공유 변수의 미분 원칙과 연결해 논하시오.

### 핵심 키워드

- `$h_t = f_W(h_{t-1}, x_t)$` · `same function, same parameters`
- `Unfold 시 여러 블록이지만 실제 $W$는 1개`
- `Arbitrary Length` · `테스트 시 길이 무관`
- `$\partial L / \partial W = \sum_t \partial L_t / \partial W$` · `Chain Rule + 공유 변수`

### 모범답안

(1) 파라미터 공유 메커니즘

RNN의 상태 전이는 단일 함수로 표현된다.

$$h_t = f_W(h_{t-1}, x_t)$$

여기서 핵심 원칙은 "The same function and the same set of parameters are used at every time step"이다. $t = 1, 2, 3, \ldots$ 어느 시점이든 동일한 함수 $f_W$를 반복 호출하며, 함수 내부의 가중치 행렬 $W_{xh}$(입력→은닉), $W_{hh}$(은닉→은닉), $W_{hy}$(은닉→출력)는 모든 시점이 같은 메모리 주소를 공유한다. 시간축으로 펼친 Unfold 그림에서는 여러 블록처럼 그려지지만, 그건 시각화일 뿐 실제로는 단 하나의 $W$를 $T$번 사용하는 것이다. 마치 같은 도장을 시간에 걸쳐 여러 번 찍는 것과 같다.

이 공유 덕분에 파라미터 수가 시퀀스 길이와 무관하게 일정하다. 100-step 시퀀스든 1000-step 시퀀스든 학습할 가중치 수는 동일하다. 또한 State Vector $h_t$가 "과거의 모든 히스토리가 압축된 인코딩 메모리"로 작동하면서 동일 함수가 시간을 따라 흐를 수 있게 한다.

(2) 공유하지 않을 때의 문제

만약 시점마다 $W_1, W_2, \ldots, W_T$를 따로 두면 두 가지 치명적 결함이 생긴다.

첫째, 파라미터 수가 $T$배로 불어난다. 시퀀스가 길수록 모델이 비대해지고 학습 데이터에 비해 과적합이 심해진다. 둘째, 그리고 더 본질적으로, Arbitrary Length(임의 길이) 처리 능력이 무너진다. 학습 시 시퀀스 길이가 $T=50$이라면 $W_1 \sim W_{50}$만 학습된다. 그러나 테스트 시 $T=100$짜리 시퀀스가 들어오면 $W_{51} \sim W_{100}$이 존재하지 않아 모델이 그냥 적용 불가능해진다. 즉, 학습 때 본 길이를 넘어서는 시퀀스에는 추론조차 못 한다.

이는 RNN의 가장 중요한 본질을 무너뜨린다. 음성 인식·기계 번역·텍스트 생성 같은 모든 시퀀스 태스크에서 입력 길이는 매번 다르다. 이 가변성을 다룰 수 있다는 것이 RNN이 CNN (Convolutional Neural Network)·MLP (Multi-Layer Perceptron) 대비 가지는 핵심 차별점인데, 파라미터 공유를 포기하면 그 차별점이 사라진다. 반대로 $W$를 공유하면 시퀀스를 몇 번 반복하든 동일 $W$를 재사용하므로 길이 제약이 자연스럽게 사라진다.

(3) BPTT에서 Gradient가 합산되는 이유

총 손실은 보통 각 시점 손실의 합이다.

$$L = \sum_{t=1}^{T} L_t$$

$W$가 모든 시점에서 공유되므로, Chain Rule에 의해 $W$에 대한 gradient는 각 시점 기여분의 합이 된다.

$$\frac{\partial L}{\partial W} = \sum_{t=1}^{T} \frac{\partial L_t}{\partial W}$$

직관적으로, "하나의 $W$가 시점 1에서도, 시점 2에서도, …, 시점 $T$에서도 쓰였으니 각 시점에서 $W$가 기여한 gradient를 모두 더해주어야 올바른 업데이트 방향이 나온다." 이는 Computational Graph의 일반 원칙인 "공유된 변수의 gradient는 그 변수가 등장한 모든 지점의 편미분을 합산한다"와 정확히 일치한다.

각 항 $\partial L_t / \partial W$를 자세히 풀면 다시 Chain Rule이 적용되어 시점 $t$에서 $W$를 거쳐 시점 1까지 거슬러 올라가는 모든 경로의 곱셈이 등장하며, 여기서 $W_{hh}$가 반복 곱해진다. $W_{hh}$의 가장 큰 특이값이 1보다 크면 폭발하고 1보다 작으면 소실되는 것이 바로 RNN의 Vanishing/Exploding Gradient 문제다. 이 문제가 LSTM (Long Short-Term Memory)·GRU (Gated Recurrent Unit) 도입의 동기가 되었다.

### 자주 빼먹는 함정

- "파라미터를 공유한다"만 쓰고 왜 공유해야 하는지(Arbitrary Length 처리)를 놓침
- Gradient 합산이 단순 합이 아니라 "공유 변수의 미분 = 등장 지점들의 편미분 합"이라는 일반 원칙임을 빠뜨림
- BPTT 수식 전개에서 $W_{hh}$가 반복 곱해진다는 사실 미언급 → 다음 주 LSTM 동기와 연결 안 됨
- 시점마다 다른 $W$를 두는 가설을 단순히 "파라미터 늘어남"으로만 답하고, 학습 길이를 넘어서는 테스트 시퀀스에 적용 불가능하다는 핵심 결함을 놓침

## 문제 7 (W8) — LSTM Cell State와 3-Gate · GRU와의 차이

### 문제

> Vanilla RNN (Recurrent Neural Network)은 Vanishing Gradient와 Long-Term Dependency 문제를 안고 있다.
>
> - (1) LSTM (Long Short-Term Memory)의 Cell State $C_t$가 Hidden State $h_t$와 별도로 도입된 이유와, Forget·Input·Output 3개 Gate가 각각 어떤 역할을 하는지 수식과 함께 설명하시오.
> - (2) Cell State가 어떻게 Vanishing Gradient를 해결하는지 BPTT (Back-Propagation Through Time) 시 gradient가 흐르는 경로를 분석하시오.
> - (3) GRU (Gated Recurrent Unit)와 LSTM의 차이점을 Cell State 유무, 게이트 개수, Update/Reset Gate 역할 관점에서 비교하고, 실무에서 어느 쪽을 먼저 시도해야 하는지 교수님 권장사항과 함께 서술하시오.

### 핵심 키워드

- LSTM: `$C_t$ = 정보 고속도로` · `$h_t$ = 외부 상호작용`
- Forget: `$f_t = \sigma(W_f [h_{t-1}, x_t])$` · `버릴 비율`
- Input: `$i_t \odot \tilde{C}_t$` · `새로 받을 비율 × 후보`
- Output: `$h_t = o_t \odot \tanh(C_t)$`
- Vanishing 해결: `$C_{t-1} \to C_t$ 경로는 element-wise 곱·덧셈만, 행렬 곱 없음` · `Uninterrupted Gradient Flow`
- GRU: `2 게이트` · `$h_t = (1-z_t) h_{t-1} + z_t \tilde{h}_t$` · `Update + Reset`
- 실무: `먼저 GRU → 부족하면 LSTM`

### 모범답안

(1) LSTM의 Cell State와 3-Gate 메커니즘

Vanilla RNN은 단일 상태 $h_t$가 메모리도 출력도 다 담당하느라 짐이 너무 무겁다. LSTM은 메모리 전용의 Cell State $C_t$와 외부와 상호작용하는 Hidden State $h_t$를 분리한다. Cell State는 교수님 표현으로 "정보를 시퀀스 끝까지 안전하게 운반하는 컨베이어 벨트(고속도로)"이고, 게이트들이 그 위에 정보를 더하거나 빼는 역할을 한다.

3개 게이트는 다음과 같다.

Forget Gate: 과거 $C_{t-1}$에서 무엇을 버릴지 결정한다.

$$f_t = \sigma(W_f \cdot [h_{t-1}, x_t] + b_f)$$

Sigmoid라 0~1 사이 값을 뽑으며, 0에 가까우면 "싹 다 잊어라", 1에 가까우면 "다 유지하라"는 신호다.

Input Gate와 Candidate: 새 정보를 얼마나, 어떤 방향으로 받을지를 분리해 결정한다.

$$i_t = \sigma(W_i \cdot [h_{t-1}, x_t] + b_i), \quad \tilde{C}_t = \tanh(W_C \cdot [h_{t-1}, x_t] + b_C)$$

$i_t$(0~1)는 비율이고 $\tilde{C}_t$(-1~+1)는 방향과 크기를 가진 후보값이다. Sigmoid와 Tanh의 역할 분리가 LSTM의 묘미다.

Cell State 업데이트는 두 항의 단순 합이다.

$$C_t = f_t \odot C_{t-1} + i_t \odot \tilde{C}_t$$

첫째 항은 과거를 Forget Gate로 거른 결과, 둘째 항은 새 후보를 Input Gate로 거른 결과. 곱셈(걸러내기)과 덧셈(추가)만으로 메모리를 업데이트한다.

Output Gate: Cell State를 외부로 얼마나 노출할지 결정한다.

$$o_t = \sigma(W_o \cdot [h_{t-1}, x_t] + b_o), \quad h_t = o_t \odot \tanh(C_t)$$

이렇게 분리된 $h_t$가 다음 레이어 또는 출력으로 흘러나간다. Cell State는 "내부 메모리 전용"으로 보호받고, 외부 노출은 Output Gate가 관리한다.

(2) Cell State와 Vanishing Gradient 해결

Vanilla RNN의 Vanishing Gradient는 BPTT에서 동일 가중치 행렬 $W_{hh}$가 시점마다 반복 곱해지면서 1보다 작은 특이값이 누적해 0에 수렴하는 데서 온다.

LSTM의 Cell State 라인은 이 곱셈 구조를 회피한다. $C_{t-1} \to C_t$로 가는 경로는

$$C_t = f_t \odot C_{t-1} + i_t \odot \tilde{C}_t$$

라는 element-wise 곱과 덧셈만 포함한다. 행렬 $W$가 곱해지지 않는다. 역전파 시점에서 보면

$$\frac{\partial C_t}{\partial C_{t-1}} = f_t$$

라는 element-wise 곱셈일 뿐이다. 강의 슬라이드는 이를 "Backpropagation from $C_t$ to $C_{t-1}$ only elementwise multiplication by $f$, no matrix multiply by $W$"라고 강조하며 "Uninterrupted Gradient Flow"라는 표현을 쓴다. 즉 Forget Gate가 1에 가깝게 열리는 한 gradient는 시퀀스를 거슬러 끊김 없이 흐른다. 교수님은 이를 ResNet (Residual Network)의 Skip Connection $H(x) = F(x) + x$와 유사하다고 비유했다 — 둘 다 "정보를 변환 없이 통과시키는 길"을 만들어 주는 철학이다.

(3) GRU와 LSTM 비교 · 실무 권장

GRU는 LSTM을 단순화한 모델로, Cell State를 없애고 Hidden State 하나에 메모리와 출력을 통합했다. 게이트도 3개에서 2개로 줄었다.

$$z_t = \sigma(W_z [h_{t-1}, x_t]), \quad r_t = \sigma(W_r [h_{t-1}, x_t])$$
$$\tilde{h}_t = \tanh(W [r_t \odot h_{t-1}, x_t]), \quad h_t = (1-z_t) \odot h_{t-1} + z_t \odot \tilde{h}_t$$

Update Gate $z_t$는 과거 $h_{t-1}$과 새 후보 $\tilde{h}_t$ 사이를 인터폴레이션한다. $z_t \to 0$이면 과거를 그대로 복사해 여러 타임스텝 정보 보존이 가능하고(Less vanishing), $z_t \to 1$이면 새 정보로 대체한다. Reset Gate $r_t$는 새 후보를 만들 때 과거 hidden state를 얼마나 살려둘지 결정한다. $r_t \to 0$이면 과거 무시(단기 의존성), $r_t \to 1$이면 다 살림(장기 의존성).

| 비교 항목 | LSTM | GRU |
|---|---|---|
| Cell State $C_t$ | 있음 (메모리 전용) | 없음 ($h_t$로 통합) |
| 게이트 개수 | 3개 (Forget, Input, Output) | 2개 (Update, Reset) |
| 파라미터 수 | 많음 | 약 25% 적음 |
| 학습 속도 | 느림 | 빠름 |
| 복잡한 장기 의존성 | 더 강함 | 다소 약할 수 있음 |
| 학습 안정성 | 매우 좋음 | 좋음 |

교수님의 실무 권장사항은 명확하다. "새로운 태스크에 메모리 모델을 적용한다면 먼저 GRU를 시도하라. 파라미터가 적고 심플해 빠른 베이스라인을 만들기 좋다. GRU가 부족하거나 더 강한 robust함이 필요하면 LSTM으로 옮겨가라." 이는 단순함이 우선이라는 모델 선택 일반 원칙과도 부합한다.

마지막으로, 두 모델 모두 게이트가 닫히는 구조 대신 "필요할 때 정보를 그대로 통과시키는" 철학을 공유하며, 이는 ResNet의 Skip Connection, Highway Network, 나아가 Transformer의 Residual + LayerNorm까지 이어지는 깊은 학습 패러다임의 공통 원리다.

### 자주 빼먹는 함정

- 게이트 활성화 함수에서 Sigmoid(게이트, 0~1)와 Tanh(후보, -1~+1)의 역할 분리를 미언급
- Vanishing 해결을 "게이트가 있어서"라고만 답하고 Cell State 라인의 element-wise 곱·덧셈 only라는 핵심 메커니즘을 놓침
- GRU와 LSTM의 게이트 개수 차이만 적고, 실무 권장(GRU 먼저)을 빼먹음
- Cell State $C_t$와 Hidden State $h_t$의 분리 의의(메모리 전용 vs 외부 상호작용)를 단순히 "두 개 있다"로 끝냄

## 문제 8 (W9) — RNN/LSTM/GRU CIFAR-10 적용 · 성능 비교 · Image Patch (ViT 빌드업)

### 문제

> 9주차 실습에서 CIFAR-10에 RNN (Recurrent Neural Network)/LSTM (Long Short-Term Memory)/GRU (Gated Recurrent Unit)를 적용했다.
>
> - (1) `(B, 3, 32, 32)` 이미지를 RNN의 시퀀스 입력 `(B, seq_len, feat)`로 변환하는 코드 한 줄 `images.permute(0, 2, 3, 1).contiguous().view(B, 32, -1)`을 단계별로 설명하고, 결과 shape이 왜 `(B, 32, 96)`이 되는지 서술하시오. `.contiguous()`가 필요한 이유도 함께 설명하시오.
> - (2) 동일 조건에서 RNN, LSTM, GRU의 Test Accuracy가 각각 약 25%, 60%, 62%로 큰 차이를 보인 이유를 장기 의존성과 게이트 메커니즘 관점에서 분석하시오.
> - (3) 행 단위 시퀀스 대신 4×4 Image Patch로 자르는 방식의 코드(`unfold` 사용)와 결과 shape `(B, 64, 48)`을 설명하고, 성능이 비슷함에도 이 방식을 다루는 이유를 Vision Transformer(ViT) 빌드업 관점에서 서술하시오.

### 핵심 키워드

- `permute(0, 2, 3, 1)` → `(B, 32, 32, 3)`
- `view(B, 32, -1)` → `(B, 32, 96)`
- `.contiguous()` 필수 이유: `permute 후 메모리 비연속`
- RNN 25%: `장기 의존성 부족` · 32행 보면 첫 행 잊음
- LSTM/GRU 60%대: `게이트 보존` · `Cell State`
- Patch: `unfold(2, 4, 4).unfold(3, 4, 4)` → `(B, 3, 8, 8, 4, 4)` → `(B, 64, 48)`
- ViT 빌드업: `이미지를 패치 시퀀스로 취급`

### 모범답안

(1) 이미지 → 시퀀스 변환 한 줄 분해

CIFAR-10 배치 텐서의 원래 shape은 `(B, C=3, H=32, W=32)`로 PyTorch의 표준 CHW 순서다. RNN은 `(B, seq_len, feat)` 형태를 받으므로 변환이 필요하다.

```python
images = images.permute(0, 2, 3, 1).contiguous().view(B, 32, -1)
```

단계별로 보면 다음과 같다.

- `permute(0, 2, 3, 1)`: 차원 순서를 (B, C, H, W)에서 (B, H, W, C)로 재배열한다. 결과 shape은 `(B, 32, 32, 3)`. 즉 채널이 마지막으로 옮겨져 NumPy의 이미지 표현(HWC)과 같아진다.
- `.contiguous()`: 메모리 레이아웃을 연속된 형태로 다시 정렬한다. `permute`는 실제 데이터 복사 없이 stride만 바꾸는 view 연산이라 메모리상에서는 텐서가 비연속(non-contiguous) 상태가 된다. `view`는 연속 메모리만 다룰 수 있어 비연속 텐서에 적용하면 RuntimeError가 난다. 따라서 `permute` 다음에는 거의 항상 `.contiguous()`가 따라온다.
- `.view(B, 32, -1)`: 텐서를 `(B, 32, ?)` 모양으로 reshape하며, `-1`은 PyTorch가 알아서 채워 넣으라는 의미다. 남은 원소 수가 $32 \times 3 = 96$이므로 결과는 `(B, 32, 96)`.

해석하면 "각 가로 한 줄(row)이 한 timestep, 그 줄의 모든 색채널 픽셀(W × C = 96차원)이 매 step의 input feature"가 된다. 즉 timestep 32, input_dim 96의 시퀀스로 이미지를 본다.

(2) 모델별 성능 차이 분석

행 단위로 시퀀스를 만들어 동일 조건에서 학습했을 때 강의에서 관찰된 Test Accuracy는 RNN ≈ 25%, LSTM ≈ 60%, GRU ≈ 62%다. 차이가 35%p 가까이 난 이유는 명확하다.

이미지의 본질은 "좌상~우하 픽셀 간의 공간적 관계"가 클래스를 결정한다는 데 있다. 32-step 시퀀스로 풀어 넣으면 RNN은 마지막 32번째 행을 처리할 시점에 1번째 행 정보를 거의 다 잃는다. 이는 7주차에서 배운 Vanishing Gradient + 고정 차원 hidden state 한계의 직격탄이다. 조교 표현으로 "RNN은 이전에 있던 거는 사실 기억을 잘 못할 테니까 롱텀이 되겠죠. 이미지 하나를 보기 위해서는. 그래서 RNN이 학습이 안 된 겁니다." 결과는 거의 무작위 추측 수준의 25%(랜덤 10%보단 낫지만 본질적으로 첫 절반 정보가 사라진 채로 분류).

LSTM은 Cell State $C_t$가 게이트로 통제된 element-wise 곱·덧셈만으로 흐르며 Uninterrupted Gradient Flow를 보장한다. Forget Gate가 적절히 열린 상태에서 첫 행 정보를 32번째 행까지 끌고 갈 수 있다. GRU도 Update Gate $z_t$가 0에 가까울 때 과거 hidden state를 그대로 복사해 보존할 수 있어 같은 효과가 난다. 둘 다 게이트로 장기 의존성을 보존해 60%대로 도약한다. GRU가 LSTM보다 약간 더 좋은 것은 파라미터가 적어 같은 데이터로 더 안정적으로 학습된 결과로 해석할 수 있으나 본질적인 차이는 거의 없다.

결론적으로 RNN 계열을 이미지에 적용하는 건 좋은 방법이 아니다. 그럼에도 실습한 이유는 다음 회차의 Vision Transformer 빌드업이라는 교육적 의도다.

(3) Image Patch와 ViT 빌드업

행 단위 자르기의 한계를 극복하기 위해 이미지를 4×4 패치 단위로 쪼개 시퀀스로 만들 수 있다.

```python
def create_patches(images, patch_size):
    B, C, H, W = images.size()  # (B, 3, 32, 32)
    patches = images.unfold(2, patch_size, patch_size).unfold(3, patch_size, patch_size)
    # → (B, 3, 8, 8, 4, 4)
    patches = patches.contiguous().view(B, C, -1, patch_size, patch_size)
    patches = patches.permute(0, 2, 1, 3, 4).contiguous()
    patches = patches.view(B, -1, C * patch_size * patch_size)
    # → (B, 64, 48)
    return patches
```

`unfold(dim, size, step)`은 sliding window 방식으로 해당 차원에서 patch를 잘라내는 연산이다. 32×32 이미지를 4×4 패치로 자르면 H 방향 8개, W 방향 8개로 총 64개 패치가 만들어지고, 각 패치는 $C \times 4 \times 4 = 48$ 차원 벡터로 펼쳐진다. 결과는 sequence length 64, feature 48의 텐서 `(B, 64, 48)`이다.

이 방식의 패치 GRU 성능은 약 62.15%로 행 단위 GRU(61.62%)와 거의 같다. 즉 RNN 계열로는 패치로 잘라도 큰 이득이 없다. 그렇다면 왜 굳이 패치 방식을 다룰까?

답은 Vision Transformer (ViT) 빌드업이다. ViT는 이미지를 16×16 (또는 그 외 크기) 패치로 잘라 시퀀스로 취급한 뒤 Transformer로 처리하는 모델이다. CNN (Convolutional Neural Network) 없이도 ImageNet에서 SOTA를 내며, 이미지 처리의 패러다임을 패치 시퀀스 + Self-Attention으로 옮겨 놓았다. 9주차 실습에서 패치 방식을 미리 경험하는 이유는 두 가지다.

첫째, ViT의 입력 전처리(이미지 → 패치 시퀀스)와 동일한 형태를 손에 익히기 위해서다. 둘째, "공간적 근접성을 가진 정보 단위로 시퀀스를 만든다"는 발상 자체가 비전 분야의 새로운 표준이 되었음을 직접 실험으로 보여주기 위해서다. 조교 표현으로 "사실 여기서 패치로 나눈다고 해서 크게 잘 되진 않죠. 하지만 다음에 실습하게 될 비전 트랜스포머 모델에서 이미지가 패치로 들어가게 됩니다. 그런 거에 기반한 어떠한 공간감이 있는 정보를 주기 위한 생각들이라고 할 수 있습니다." 즉 RNN 계열의 한계를 직접 경험하면서 동시에 다음 단계(Transformer)로 가는 다리를 놓는 실습이다.

### 자주 빼먹는 함정

- `.contiguous()`를 빼먹고 설명. `permute` 후 메모리 비연속이라 `view`가 안 된다는 점이 출제 단골
- `(B, 32, 96)`에서 96이 어떻게 나오는지(W × C = 32 × 3) 명시 안 함
- RNN 25% 답을 단순히 "성능이 낮다"라고만 적고 32-step 시퀀스에서 첫 행 정보 소실이라는 구체적 메커니즘을 놓침
- 패치 실습의 목적이 ViT 빌드업이라는 핵심을 빠뜨리고 "성능 비슷하다"로 끝냄
