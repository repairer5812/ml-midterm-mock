# 기계학습2주차

## 1. 인공지능(AI)과 머신러닝(ML)의 개요 및 역사적 배경

### 1-1. 인공지능(Artificial Intelligence)의 정의

- 개념: 컴퓨터 시스템이 인간의 지능(Human Intelligence)을 모방한 모델.
- 인간의 지능이란?: 학습, 추론, 문제 해결(Problem Solving), 의사 결정(Decision Making), 인식(Perception) 등의 활동을 의미.

### 1-2. 전문가 시스템 (Expert System) - 머신러닝 이전의 지배적 모델

- 개념: 모델 자체를 '전문가'로 만드는 방식. 도메인 전문가로부터 지식이나 규칙을 추출하여 시스템에 직접 프로그래밍해 넣음(Knowledge Base).
- 작동 방식: 주로 규칙 기반(Rule-based) 엔진을 사용. 프로그래밍의 If-then 구문(어떤 조건을 만족하면 어떤 액션을 취해라)을 통해 전문가의 로직을 구현.
- 대표적 예시: MYCIN (마이신) - 1970년대 개발된 초창기 의료 전문가 시스템으로, 박테리아 감염을 진단하고 치료법을 추천함. 간단한 챗봇 등에도 적용됨.

💡 핵심 출제 포인트: 전문가 시스템의 한계점 (머신러닝이 등장한 이유)

- 유연성(Flexibility) 부족: 이미 정해진 룰(Predefined rules)에 고정되어 있어, 전문가조차 예상치 못한 상황이나 예외적인 인풋이 들어오면 모델이 이상하게 작동함.
- 환경 변화 적응 불가: 환경이 변하거나 새로운 패턴이 등장하면 시스템이 스스로 적응하지 못하고, 사람이 수동(Manually)으로 룰을 일일이 수정해야 함 (비용과 시간이 많이 듦).
- 확장성(Scalability) 부족: 도메인이 복잡해질수록 필요한 규칙의 수가 기하급수적으로 늘어나 유지보수가 매우 어려워짐(Brittle).

### 1-3. 머신러닝 (Machine Learning)

- 개념: AI의 하위 분야(Subfield)로, 규칙을 직접 입력하는 대신 데이터(Data)에 기반하여 컴퓨터가 스스로 학습하고 의사결정을 내리도록 하는 알고리즘 및 모델.
- 장점: 전문가 시스템의 한계와 정확히 반대. 유연성(Flexibility)과 확장성(Scalability)이 높아 복잡하고 동적인 환경에서도 데이터가 쌓일수록 잘 적응함.

## 2. 머신러닝의 3가지 핵심 요소 (인지 과정과의 비교)

사람의 인지 과정(뇌)과 머신러닝 모델을 비교. 머신러닝 시스템을 다룰 때는 항상 다음 3가지를 고려해야 합니다.

1. 데이터 (Data):
    - 지식의 종류와 내용. 지식을 추출할 수 있는 관측치.
    - 사람에 비유하면: 뇌가 학습하는 과정에서 겪는 '경험'. 머신러닝은 데이터로부터 자동 학습하므로 데이터의 존재가 필수적임.
2. 알고리즘 (Algorithm):
    - 데이터를 처리하고 유용한 패턴을 찾아내는 절차나 함수 (소프트웨어적 측면). 목적 함수에 따라 종류가 달라짐.
    - 사람에 비유하면: 뇌의 '학습 방법'. 동일한 데이터(경험)가 주어져도 사람마다 효율적으로 빠르게 처리하거나 비효율적으로 느리게 처리할 수 있음.
3. 모델 (Model):
    - 알고리즘에 의해 학습된 결과물로, 입력(Input)을 출력(Output)으로 매핑(Mapping)하는 아키텍처.
    - 사람에 비유하면: 뇌의 '인지 단계/방법'.

💡 [Q&A 디테일] 모델과 알고리즘의 차이점:

- 모델: 눈으로 '물통'의 빛 정보(픽셀값, X)가 들어왔을 때, 뇌에서 처리 과정을 거쳐 "이것은 물통(Y)이다"라고 결론 내리는 매핑 함수(Function).
- 알고리즘: 빛 정보가 뇌의 고차원 인지 영역까지 도달하기 위해 거치는 수많은 처리 과정(프로세스). 목적(단순 명명 vs 후기 작성 등)에 따라 학습 방식이 달라짐.

## 3. 머신러닝의 3대 주요 학습 방법

### 3-1. 지도 학습 (Supervised Learning)

- 개념: 데이터와 함께 정답(Label)이 주어지는 학습. 감독자(Supervisor)가 있는 형태.
- 유형 1: 분류 (Classification)
    - 새로운 데이터의 카테고리(이산적인 클래스 라벨)를 예측.
    - 이진 분류 (Binary): 클래스가 2개. (예: 스팸 vs 정상 메일, 질병 유무)
    - 다중 분류 (Multi-class): 클래스가 여러 개. (예: 이미지 내 객체가 개, 고양이, 자동차 중 무엇인지 분류, 알파벳 26개 분류)
- 유형 2: 회귀 (Regression)
    - 설명 변수(X)와 결과값(Y)의 관계를 모델링하여 연속적인 수치(Continuous outcome)를 예측.
    - 교수님 예시:
        - 집값 예측 (House price estimation): 시간축이 아닌 방 개수, 층수, 위치 등 다양한 팩터(X)를 기반으로 연속적인 집값(Y) 예측.
        - 아이 성장 모델링: 나이에 따른 키/몸무게 변화 예측.
        - 의료 측정: 음식 섭취량에 따른 혈당 수치 등 연속적 진단값 예측.

💡 [Q&A 디테일] 회귀와 분류의 구분 기준:

회귀 문제도 분류처럼 풀 수 있지 않냐는 질문에 대해: 치역(결과값, Y)이 연속적(Continuous)이면 회귀, 이산적(Categorical)이면 분류에 속함. 회귀 문제를 억지로 분류로 풀면 모델 학습이 이상해질 수 있음. (통계학에서도 회귀와 상관관계를 많이 다룸)

### 3-2. 비지도 학습 (Unsupervised Learning)

- 개념: 정답(Label)이 없는 상태에서 데이터만 보고 숨겨진 패턴이나 구조를 스스로 파악하는 학습.
- 유형 1: 군집화 (Clustering)
    - 사전 지식 없이 데이터의 특성(Feature - 예: 색깔, 형태 등)을 보고 유사도에 근거해 의미 있는 그룹으로 묶는 작업.
    - 교수님 예시: 친구 관계 그룹핑(친한 친구, 애매한 친구 등 스스로 기준 설정), 레스토랑 메뉴 클러스터링(재료, 칼로리 기준), 고객 세그먼테이션(구매 빈도, 소비액 기준).
- 유형 2: 차원 축소 (Dimensionality Reduction)
    - 우리가 사는 세상의 고차원 데이터(예: 수많은 사람의 색깔, 형태, 성별 등 복잡한 정보)에서 노이즈를 제거하고 중요한 핵심 특징만 뽑아 저차원으로 압축하는 기법.
    - 목적: 데이터 이해도 증가, 연산 효율성 확보, 2D/3D 시각화 가능.

### 3-3. 강화 학습 (Reinforcement Learning)

- 개념: 정답(Label)이 주어지지는 않지만, 에이전트(Agent)가 환경과 상호작용하며 스스로 취한 행동(Action)에 대해 보상(Reward)이나 처벌(Punishment)을 받고, 이를 통해 최적의 전략을 시행착오로 학습.

💡 [Q&A 디테일] 공장 로봇 영상이 왜 강화학습인가?:

정해진 레이블(여기엔 밥, 저기엔 반찬)을 딱 주는 지도학습과 달리, 기계가 스스로 액션을 취해보고 "이건 틀렸어(처벌), 이건 맞아(보상)"라는 피드백 과정을 수십 번 반복하면서 규칙을 스스로 학습하기 때문.

*(참고) 신경망 (Neural Networks): 사람의 뇌 뉴런 구조와 학습 방식을 모방한 머신러닝의 하위 분야(서브필드). 딥러닝도 결국 머신러닝의 로직과 철학을 공유함.*

## 4. 일반화 (Generalization)와 데이터 분할

### 4-1. 일반화 (Generalization)의 개념

- 정의: 훈련에 사용되지 않은 새로운 데이터(Unseen data, Test set)가 주어졌을 때 모델이 얼마나 정답을 잘 예측하고 작동하는가를 나타내는 능력.
- 중요성: 머신러닝의 가장 핵심적인 과제(Challenge). 훈련 데이터를 아무리 100% 다 맞추더라도, 새로운 데이터에서 성능(Generalization Performance)이 낮으면 실패한 모델임.

### 4-2. 데이터 분할 (Data Split)

모델의 일반화 성능을 평가하기 위해 우리가 가진 전체 데이터를 보통 3가지로 나눕니다.

1. Training set (훈련 세트): 모델을 학습시키고 데이터 내의 패턴을 캡처하는 데 사용.
2. Validation set (검증 세트):
    - 개발 과정 중 모델을 선택하거나 파라미터(Parameter)를 튜닝할 때 가이드 역할을 함.
    - 테스트 세트를 미리 볼 수 없으므로, 훈련 중 일반화 성능(Generalization Error)을 대략적으로 추정(Estimate)하기 위해 사용.
3. Test set (테스트 세트): 훈련 및 검증이 완전히 끝난 후, 실제 세계(Real-world)의 데이터에서 모델이 어떻게 행동할지 최종적으로 평가하기 위해 철저히 격리해 두는 세트.

## 5. 교차 검증 (Cross-Validation)

단순히 데이터를 한 번만 나누면(Train/Validation) 우연히 특정 데이터만 검증 세트로 빠져 성능 평가가 왜곡될 수 있습니다. 이를 방지하기 위한 체계적인 방법입니다.

### 5-1. K-fold Cross-Validation (K-폴드 교차 검증)

- 방식: 전체 데이터를 동일한 크기의 K개 부분(Fold)으로 나눔.
- 과정: 총 K번의 훈련(Iteration)을 반복. 매 반복마다 K-1개의 폴드는 훈련용으로, 나머지 1개의 폴드는 검증용으로 번갈아 가며 사용.
- 결과: K번의 검증 성능(Validation Score)을 평균 내어 최종 모델 성능을 평가함.

### 5-2. Leave-one-out Cross-Validation

- 방식: K-fold의 극단적인 케이스로, K가 전체 데이터의 개수(N)와 같음.
- 과정: 딱 1개의 데이터 포인트만 Validation으로 남기고, 나머지 모든 데이터를 Training에 사용. 이를 데이터 개수만큼 반복.
- 특징: 폴드 수가 너무 많아져 연산 비용이 크기 때문에 실무에서는 잘 사용하지 않음.

### 5-3. 💡 Stratified K-fold Cross-Validation (핵심 출제 포인트)

- 문제 상황: 데이터의 클래스 비율이 심하게 불균형(Imbalanced)할 때 발생.
    - *예시:* Class 1이 10개, Class 2가 200개일 때 단순 K-fold를 쓰면, 어떤 폴드에는 Class 1이 아예 안 들어갈 수도 있음.
- 해결책: 각 폴드(Fold)를 나눌 때, 원본 데이터의 클래스 비율(Proportion)을 동일하게 유지(Preserve)하도록 분할하는 변형된 교차 검증 기법.

## 6. 과소적합(Underfitting) vs 과적합(Overfitting) ★가장 중요★

머신러닝 모델을 평가할 때 부딪히는 두 가지 주요 문제입니다.

### 6-1. 과소적합 (Underfitting)

- 개념: 모델이 훈련 데이터의 패턴조차 충분히 학습하지 못한 상태.
- 원인: 모델의 용량(Capacity)이 너무 낮거나(예: 복잡한 곡선 데이터에 단순한 1차 함수 직선 모델을 사용), 훈련(Epoch)을 너무 적게 시켰을 때 발생.
- 결과: Training Error 자체가 매우 높게 나옴.

### 6-2. 과적합 (Overfitting)

- 개념: 모델이 훈련 데이터에 너무 과도하게 맞춰져(꽉 피팅되어), 훈련 데이터 내의 불필요한 노이즈(Noise)와 아웃라이어까지 전부 암기(Memorize)해버린 상태.
- 원인: 모델의 용량(Capacity)이 너무 크거나, 훈련을 너무 많이 시켰을 때 발생.
- 결과: Training Error는 거의 0에 가깝지만, 새로운 데이터(Test set)가 들어오면 예측이 크게 빗나가 Validation Error가 급증함. (일반화 실패)
- 해결책: Validation Error가 감소하다가 다시 상승하기 직전의 지점(Sweet spot)에서 훈련을 멈추거나(Early Stopping), 모델의 복잡도를 줄여야 함.

### 6-3. Bias-Variance Tradeoff (편향-분산 트레이드오프)

과소/과적합을 통계적 용어로 설명한 것으로, 이 둘을 동시에 최소화하는 것은 불가능하다는 딜레마입니다.

- Bias (편향): 학습 알고리즘의 잘못된, 혹은 너무 단순한 가정으로 인해 발생하는 에러.
    - *특징:* 모델 Capacity를 늘릴수록 Bias Error는 떨어짐. (High Bias = Underfitting)
- Variance (분산): 훈련 데이터의 작은 변동성(노이즈)에 모델이 너무 민감하게 반응하여 발생하는 에러.
    - *특징:* 모델 Capacity를 늘릴수록 Variance Error는 올라감. (High Variance = Overfitting)

### 6-4. 💡 최신 연구 동향: Deep Double Descent (오픈AI 발견)

- 전통적 지식: 모델 크기나 Epoch가 증가하면 Validation Error가 U자형 곡선을 그리며 다시 상승(Overfitting)하므로 멈춰야 함.
- 새로운 발견: OpenAI의 연구진이 모델 크기나 훈련(Epoch)을 멈추지 않고 계속 무작정 늘려보았더니, 상승하던 Error가 어느 순간 다시 하강(Descent)하여 기존보다 훨씬 더 낮아지는 현상을 발견함.
- 교수님 코멘트: 스케일링 법칙(Scaling Law, 파라미터와 데이터를 무작정 때려 박으면 성능이 증가하는 현상 - 예: GPT 1,2,3,4)과 관련이 깊음.

## 7. OOD (Out-of-Distribution)와 지름길 학습 (Shortcut Learning)

머신러닝 모델이 훈련 데이터에서는 잘 작동하다가 실제 환경에서 엉뚱한 결과를 내놓는 근본적인 이유입니다.

### 7-1. 데이터 분포의 중요성

- In-distribution (분포 내 데이터): 테스트 데이터가 훈련 데이터와 유사한 특성과 패턴(분포)을 공유하는 경우. 모델이 예상대로 잘 작동함.
- Out-of-distribution (OOD, 분포 외 데이터): 훈련 데이터에는 전혀 없었던 새로운 패턴, 특징, 조건이 포함된 데이터.
    - *예시:* 횡단보도, 자동차 등 일반적인 도로 교통 표지판만 학습한 자율주행 모델에 갑자기 '새(Bird)' 사진이 등장하는 경우.
- 문제점: 사람은 뜬금없다고 판단하고 넘어가지만, 인공지능은 이를 횡단보도로 100% 확신하는 등 이상 행동(로버스트하지 못함)을 보임.

### 7-2. 💡 지름길 학습 (Shortcut Learning) - 핵심 출제 포인트

- 개념: 딥러닝 모델이 사람이 의도한 본질적이고 일반화 가능한 특징(예: 객체의 형태)을 학습하는 대신, 데이터셋에 우연히 존재하는 '쉬운 지름길(Spurious patterns)'을 힌트로 삼아 정답을 맞히는 현상.
- 교수님의 구체적 예시 1 (별과 달):
    - 사람: 별 모양은 A, 달 모양은 B라고 형태를 보고 학습함.
    - 모델: 모양이 아니라 화면 상의 '위치'를 보고 학습함. (왼쪽에 있으면 A, 오른쪽에 있으면 B). 따라서 위치가 바뀌면 전혀 엉뚱한 대답을 함.
- 교수님의 구체적 예시 2 (고양이와 코끼리):
    - 고양이 형태의 이미지에 코끼리 가죽 텍스처(질감)를 입힌 사진을 보여줌.
    - 사람: 형태를 보고 "고양이"라고 대답함.
    - 모델: 텍스처라는 지름길을 통해 학습했기 때문에 99%의 확률로 "코끼리"라고 대답함.
- 결과: 이러한 지름길 학습 때문에 모델은 OOD 데이터나, 인간 눈에는 보이지 않는 미세한 노이즈를 섞은 적대적 예제(Adversarial Example)에 매우 취약해짐.

- MYCIN 시스템의 한계와 머신러닝의 장점(유연성/확장성) 비교 서술.
- 회귀(Regression)는 결과값이 연속적, 분류(Classification)는 이산적이라는 차이점.
- 데이터 불균형 시 K-fold 대신 Stratified K-fold를 써야 하는 이유.
- Underfitting(High Bias)과 Overfitting(High Variance)의 발생 원인과 해결책(Validation curve 해석).
- 딥러닝 모델이 OOD 데이터에서 엉뚱한 결과를 내는 원인인 '지름길 학습(Shortcut Learning)'의 개념과 예시(별/달 위치, 고양이/코끼리 텍스처).
