# 📘 6차시 — 스택(Stack)과 수식의 계산

> 핵심 한 줄 요약
> 스택은 한쪽 끝(Top)에서만 삽입·삭제가 발생하는 LIFO 구조. 중간 데이터 이동이 없어 정적 구현(배열) 도 충분. 수식 계산(중위 → 후위 → 평가) 은 스택의 가장 중요한 응용이며, 이번 시험의 최고 중요 파트.

---

## 🎯 6차시 핵심 출제 포인트 프리뷰

1. ⭐ 수식 계산 파트에서만 3문제 출제 (변환, 계산 등)
2. 스택은 정적 구현이 유리하다 (중간 Shift 불필요)
3. C언어 증감 연산자: `++top` vs `top++` 차이 주의
4. 중위식 → 후위식 수작업 변환 3단계
5. 중위식 → 후위식 스택 변환 알고리즘 (icp, isp)
6. 후위식 계산 (피연산자 PUSH, 연산자 만나면 POP 2개)
7. 중첩 괄호 처리 원리 (교수님 Q&A)

## 0. 지난 시간 복습: 연결 리스트와 시간 복잡도

> 강의 초반부에 교수님께서 지난 시간에 배운 리스트의 동적 구현(연결 리스트)과 시간 복잡도에 대해 매우 상세히 리뷰. 이 부분의 빈칸 채우기 문제가 출제될 수 있다.

### 0.1 자료구조를 동적으로 구현하는 이유

- 가변성: 데이터 크기가 고정되어 있지 않음.
- 데이터 이동 방지 (가장 중요한 이유): 정적(배열)으로 연속된 기억 공간에 리스트를 구현하면, 중간에 데이터를 삽입/삭제할 때 빈칸을 만들거나 채우기 위해 Shift가 필수적.
- 알고리즘의 성능(시간 복잡도)을 높이려면 연산 횟수를 줄여야 하는데, 데이터 이동은 연산량을 크게 증가시키므로 이를 지양하기 위해 동적 구현 사용.

### 0.2 💡 각 리스트별 삽입 연산 시간 복잡도 비교 (핵심 출제)

| 자료구조 | `add_first` | `add_last` | 기타 |
|---------|------------|-----------|------|
| 파이썬 동적 배열 | $O(n)$ | Amortized $O(1)$ | 공간 재할당 시 $O(n)$, 분할 상환으로 $O(1)$ |
| 단순 연결 리스트 | $O(1)$ | $O(n)$ | Head부터 끝까지 탐색 필요 |
| 원형 연결 리스트 | $O(1)$ | $O(1)$ ✨ | 선행 노드 앞 삽입은 한 바퀴 돌아 $O(n)$ |
| 이중 연결 리스트 | $O(1)$ | $O(n)$ | — |
| 이중 원형 연결 리스트 | $O(1)$ | $O(1)$ | 선행 노드 앞/뒤 모두 $O(1)$ ⭐ 가장 많이 사용 |

> ⭐ Amortized $O(1)$ (분할 상환): 파이썬 리스트는 공간이 꽉 차면 1.2배~3배로 재할당 후 데이터 이동. 대부분의 `append`는 $O(1)$이지만 재할당 시 $O(n)$ → 평균적으로 $O(1)$로 간주.

### 0.3 이중 연결 리스트의 노드 삽입 순서 (빈칸 채우기 주의)

> 💡 교수님 강조: 노드 삽입 시 포인터 연결 순서가 끊어지지 않게 주의!

1. 새로운 노드(New Node)의 링크 필드를 먼저 설정.
2. New Node의 왼쪽/오른쪽 링크가 선행 노드(p)와 그 다음 노드를 가리키게 함.
3. 기존 노드들의 링크를 업데이트할 때, 선행 노드(p)가 가리키는 기존 다음 노드의 주소가 유실되지 않도록 순서를 신중히 결정. (순서를 잘못하면 리스트가 끊어짐)

## 1. 스택(Stack)과 큐(Queue)의 기본 개념

스택과 큐는 리스트 자료구조의 특별한 형태. 데이터 간 관계 1:1이며 순서(내 앞뒤에 누가 있는지) 가 중요.

> 🗣️ 교수님 비유: "쌍둥이는 아니고 자매 같은 관계"

### 1.1 스택 (Stack)

| 항목 | 내용 |
|------|------|
| 특징 | 삽입(Insert)·삭제(Delete)가 한쪽 끝(Top) 에서만 발생 |
| 원리 | LIFO (Last-In, First-Out) — 마지막에 들어온 데이터가 가장 먼저 나감 |
| 삽입 연산 | PUSH: Top을 1 증가시키고 그 위치에 삽입 |
| 삭제 연산 | POP: Top 위치의 데이터를 반환하고 Top을 1 감소 |

#### 스택의 활용처 (매우 중요)

| 활용 | 설명 |
|------|------|
| 함수 호출 (Call Stack) | 함수 호출 시 스택 프레임을 할당받아 쌓고, 실행이 끝나면 위에서부터 되돌아옴 |
| 수식의 계산 ⭐ | 연산자와 피연산자를 스택에 넣고 빼면서 계산 (이번 시험의 핵심) |
| 에디터의 Undo | 이전 상태를 스택에 저장, Undo 시 가장 최근 상태 Pop |
| 웹 브라우저 뒤로 가기 | 방문한 웹사이트 기록을 스택에 저장 |

### 1.2 큐 (Queue) [예고]

| 항목 | 내용 |
|------|------|
| 특징 | 삽입은 한쪽(Rear), 삭제는 반대쪽(Front) |
| 원리 | FIFO (First-In, First-Out) |
| 삽입 | Enqueue |
| 삭제 | Dequeue |

## 2. 스택 ADT 및 주요 연산

스택은 특정 위치(Top)에서만 연산이 일어나므로, 일반 리스트처럼 위치를 지정하는 `add_last`, `add_first` 같은 연산이 없다.

| 요소 | 의미 |
|------|------|
| `top` | 스택에서 원소의 삽입·삭제가 일어나는 위치를 가리키는 변수. 초기값 -1 (비어있음) |
| `PUSH(s, item)` | 삽입 |
| `POP(s)` | 삭제 + 반환 |
| `peek(s) = top(s)` 💡 | 맨 위 원소를 삭제하지 않고 반환(확인)만 함. 리스트의 `getEntry`와 유사 |
| `is_empty()` | 스택이 비어있는지 (정적 구현 시 `top == -1`). 비어있으면 POP 불가 |
| `is_full()` | 스택이 가득 찼는지 (정적 구현 시 `top == MAX_SIZE - 1`). 가득 차면 PUSH 불가 |

## 3. 스택의 구현: 정적 vs 동적

### 3.1 💡 [핵심 출제] 스택의 정적 구현 (1차원 배열 사용)

> 스택은 정적 구현이 유리하다!

#### 이유
- 스택은 PUSH/POP을 할 때 맨 마지막 원소(top)에서만 작업.
- 일반 리스트처럼 중간에 데이터 Shift가 전혀 필요 없음.
- 따라서 구현이 간단한 정적(배열) 구현을 매우 많이 사용.

#### 초기 설정
- `top = -1` (배열 인덱스는 0부터 시작하므로, 비어있음을 의미)

#### 💡 C언어 증감 연산자 주의 (메모 강조)

| 형태 | 동작 |
|------|------|
| `++변수` (전위형) | 변수를 먼저 1만큼 증가시키고 나서 연산(저장)에 사용 |
| `변수++` (후위형) | 연산(저장)을 먼저 수행하고 나서 변수를 1만큼 증가 |

스택의 PUSH 연산에서는 `stack[++top] = item;` 형식으로 사용 → `top`을 먼저 하나 증가시킨 뒤, 그 위치에 데이터를 넣는다.

#### PUSH 연산 로직
```
1. is_full()인지 검사 → 꽉 찼으면 삽입 불가
2. top을 1 증가 (++top)
3. 증가된 top 위치에 item 삽입
```
→ 코드: `stack[++top] = item;`

#### POP 연산 로직
```
1. is_empty()인지 검사 (top == -1) → 비어있으면 삭제 불가
2. 현재 top이 가리키는 위치의 데이터를 반환
3. top을 1 감소 (top--)
```
→ 코드: `item = stack[top--];`

---

### 3.2 스택의 동적 구현 (연결 리스트 사용)

크기가 제한되지 않는다는 장점, 하지만 삽입/삭제 시 메모리 할당/해제 필요.

#### 구조적 특징
- 동적 스택에서 `top` 포인터는 항상 가장 최근에 삽입된 맨 위(상단) 노드를 가리킴.
- 각 노드의 링크 필드(`link` 또는 `next`)는 자신의 아래(이전)에 있는 노드를 가리킴.

#### 시각화
```
top ──▶ [C | link] ──▶ [B | link] ──▶ [A | NULL]
         (최근)                         (최초)
```

#### 💡 PUSH 연산 (메모 강조)

```c
new_node = malloc(...);
new_node->link = top;   // ① 새 노드의 링크가 현재의 top을 가리킴 (자신의 바로 아래가 될 노드)
top = new_node;         // ② top 포인터를 새 노드로 업데이트
```

#### POP 연산

```c
if (is_empty()) error;
temp = top;             // ① 삭제할 노드의 주소를 임시 저장 (메모리 반환용 필수!)
top = top->link;        // ② top을 그 아래 노드로 이동
데이터 반환;
free(temp);             // ③ 메모리 해제
```

## 4. 💡 스택의 응용: 수식의 계산 (이번 시험 최고 중요 파트 🌟)

> ⭐ 핵심 출제 포인트: 수식 계산 파트에서만 무려 3문제 출제! (변환, 계산 등)

### 4.1 수식 표기법의 종류

| 표기법 | 형식 | 예시 | 특징 |
|--------|------|------|------|
| 중위 표기식 (Infix) | `피연산자1 연산자 피연산자2` | `3 * 5` | 일상적 방식. 우선순위·괄호 필요 → 컴퓨터가 바로 계산 어려움 |
| 전위 표기식 (Prefix = Polish) | `연산자 피연산자1 피연산자2` | `* 3 5` | 폴란드 수학자 얀 우카시에비치(Jan Łukasiewicz) 가 1800년대 초 고안 |
| 후위 표기식 (Postfix = Reverse Polish) | `피연산자1 피연산자2 연산자` | `3 5 *` | 💡 컴퓨터가 사용하는 방식 |

> 🗣️ 교수님 농담: "얀 우카시에비치... 이름이 너무 길어서 얀 교수님이라고 부르겠습니다."

#### 💡 왜 후위 표기식인가?
- 컴퓨터는 왼쪽에서 오른쪽으로 데이터를 순차적으로 읽음.
- 후위 표기식은 괄호가 필요 없고, 연산자 우선순위를 신경 쓸 필요 없이 왼쪽에서 오른쪽으로 가면서 기계적으로 계산하기에 가장 적합.

### 4.2 컴퓨터의 수식 계산 2단계 (시험 출제 포인트)

| 단계 | 작업 | 스택 역할 |
|------|------|-----------|
| 1단계 (변환) | Infix → Postfix | 연산자(Operator) 를 스택에 넣었다 뺌 |
| 2단계 (계산) | Postfix 계산 | 피연산자(Operand) 를 스택에 넣었다 뺌 |

## 5. 💡 수식 계산 1단계: 중위식 → 후위식 수작업 변환

> 시험 출제 포인트: 슬라이드 p37 연습문제 1번. 객관식 보기 4개 중 정답 고르는 문제로 출제 확률 높음.

### 5.1 수작업 변환 팁 (교수님 비법)

1. 연산자 우선순위에 맞게 수식 전체에 빠짐없이 괄호를 친다.
2. 각 괄호 안에 있는 연산자를 해당 괄호의 바로 바깥(오른쪽)으로 빼낸다.
3. 모든 연산자를 뺐다면 괄호를 모두 지운다.

### 5.2 강의 중 다룬 변환 예시 (필독)

#### 예제 1) `A / B * C`

- 우선순위가 같으면 왼쪽부터 계산 (좌결합성).
- 괄호 치기: `( ( A / B ) * C )`
- 연산자 빼기:
    - `A / B` 에서 `/`를 밖으로 → `(A B) /`
    - 그다음 `* C` 의 `*`를 맨 밖으로 → `(A B) / C *`
- 괄호 제거: **`A B / C *`** ✅

#### 예제 2) `A / B * ( C + D ) + E` (p37 연습문제)

- 괄호 치기: `( ( ( A / B ) * ( C + D ) ) + E )`
- 연산자 이동:
    - `( A B / )`
    - `( C D + )`
    - 두 개를 곱함 `*` → `( A B / C D + * )`
    - 마지막에 E와 더함 `+` → `( A B / C D + * E + )`
- 최종 결과: `A B / C D + * E +` ✅

#### 예제 3) `( ( ( A / B ) + C ) - ( D * E ) )` (p37 연습문제)

- 이미 괄호가 쳐져 있음. 안쪽부터 차례대로 연산자를 오른쪽 괄호 밖으로 뺀다.
    - `A / B` → `A B /`
    - `D * E` → `D E *`
    - `+ C` 부분 → `A B / C +`
    - 마지막 `-` 연산자 → `A B / C + D E * -`
- 최종 결과: `A B / C + D E * -` ✅

## 6. 💡 수식 계산 2단계: 스택을 이용한 후위식 변환 알고리즘 (Infix → Postfix)

> 핵심 출제 포인트: 슬라이드 p42 연습문제 및 스택의 변화 과정 추적

> 🔑 대원칙: "피연산자는 바로 출력하고, 연산자는 스택에 넣는다"

### 6.1 변환 알고리즘의 기본 규칙

| 기호 | 처리 |
|------|------|
| 피연산자 (Operand) | 입력받는 즉시 그대로 출력 |
| 연산자 (Operator) | 스택에 PUSH (단, 우선순위에 따라) |
| EOS (End of Sentence) | 수식이 끝났음을 알림. 스택에 남아있는 모든 연산자를 순차적으로 POP하여 출력 |

### 6.2 연산자 우선순위 비교 (icp vs isp)

연산자가 스택에 들어갈 때는 현재 스택의 맨 위(top)에 있는 연산자와 새로 들어올 연산자의 우선순위를 비교해야 한다.

| 약어 | 의미 |
|------|------|
| icp (Incoming Precedence) | 지금 막 읽어 들인(들어오려는) 연산자의 우선순위 |
| isp (In-Stack Precedence) | 현재 스택의 맨 위(top)에 있는 연산자의 우선순위 |

#### 🧮 비교 로직

| 조건 | 동작 |
|------|------|
| `isp < icp` | 들어오려는 연산자 우선순위가 더 크면 → PUSH |
| `isp >= icp` | 들어오려는 연산자 우선순위가 작거나 같으면 → 스택 안 연산자의 우선순위가 작아질 때까지(`isp < icp`가 될 때까지) POP하여 출력, 그 후 새 연산자 PUSH |

> ⚠️ 주의: 우선순위가 '같을 때'도 PUSH가 아니라 POP을 먼저 해야 한다!

### 6.3 💡 괄호의 마법 (`(` 와 `)`)

괄호가 있는 수식은 변환이 다소 복잡하지만, 아래 규칙만 기계적으로 적용하면 된다.

#### 🔸 왼쪽 괄호 `(`
- 들어올 때(icp)는 우선순위가 가장 높음 → 무조건 스택에 PUSH.
- 하지만 스택에 일단 들어가고 나면(isp), 우선순위가 가장 낮음 → 다음에 어떤 연산자가 들어오든 무조건 `(` 위에 쌓임 (바닥에 깔아주는 역할).

#### 🔸 오른쪽 괄호 `)`
- 이 기호를 만나면, 스택에서 왼쪽 괄호 `(` 가 나올 때까지 계속 POP하여 출력.
- 마지막에 짝을 맞춘 왼쪽 괄호 `(` 와 오른쪽 괄호 `)` 는 출력하지 않고 버림 (제거).

---

### 6.4 💡 교수님 Q&A: "이중 괄호(중첩 괄호)면 어떻게 하나요?"

> 강의 중 한 학생이 아주 중요한 질문을 했음. 시험에 응용되어 나올 수 있음!

- 학생 질문: "괄호 열고 연산하다가 또 괄호가 열리는 중첩 괄호면 어떻게 하나요? 왼쪽 괄호가 살아있다는 걸 기억해야 하나요?"
- 교수님 답변:
    > "기억할 필요 없이 기계적으로 처리됩니다.
    > 첫 번째 `(` 가 스택에 들어가면 우선순위가 가장 낮아져 바닥에 깔립니다.
    > 두 번째 `(` 가 들어올 때도 `icp`가 가장 높으므로 무조건 그 위에 PUSH됩니다. 그리고 스택 안에서 다시 우선순위가 가장 낮아지죠.
    > 이후 첫 번째 `)` 를 만나면, 스택을 위에서부터 POP하다가 가장 처음 만나는(가장 위에 있는) `(` 까지만 POP하고 멈춥니다.
    > 따라서 바닥에 깔려있던 바깥쪽 `(` 는 안전하게 살아남아 다음 `)` 를 기다리게 됩니다."

## 7. 💡 수식 계산 3단계: 스택을 이용한 후위식 계산 알고리즘 (Evaluation)

> 핵심 출제 포인트: 슬라이드 p49 연습문제 — 후위식을 계산할 때 스택의 변화

> 🔑 대원칙: "피연산자를 스택에 넣고, 연산자를 만나면 계산한다"

### 7.1 계산 알고리즘 규칙

| 기호 | 처리 |
|------|------|
| 피연산자 (Operand) | 무조건 스택에 PUSH |
| 연산자 (Operator) | 스택에서 피연산자 2개 POP → 먼저 POP된 것이 `op2`, 나중 POP된 것이 `op1` → `op1 연산자 op2` 계산 → 결과를 다시 PUSH |
| EOS | 수식이 끝나면 스택에 최종 결과값 딱 하나만 남음 → POP하면 계산 완료 |

> ⚠️ 주의: 먼저 POP 된 것이 `op2`, 나중에 POP 된 것이 `op1`! 순서가 중요 (뺄셈, 나눗셈에서 결과가 달라짐).

### 7.2 성능
- 시간 복잡도: $O(n)$ ($n$은 수식 기호의 개수, 한 번씩만 훑으면 끝)
- 공간 복잡도: $O(n)$

---

### 7.3 계산 예시 추적 (강의 및 슬라이드)

#### 예제 1) `1 2 + 7 *`

| 입력 | 처리 | 스택 상태 |
|------|------|----------|
| `1` | 피연산자 → PUSH | `[1]` |
| `2` | 피연산자 → PUSH | `[1, 2]` |
| `+` | 연산자 → POP(2), POP(1) → `1+2=3` → PUSH | `[3]` |
| `7` | 피연산자 → PUSH | `[3, 7]` |
| `*` | 연산자 → POP(7), POP(3) → `3*7=21` → PUSH | `[21]` |
| EOS | POP | 결과: 21 ✅ |

---

#### 예제 2) `6 2 / 3 - 4 2 * +`

| 입력 | 처리 | 스택 상태 |
|------|------|----------|
| `6` | PUSH | `[6]` |
| `2` | PUSH | `[6, 2]` |
| `/` | POP 2, POP 6 → `6/2=3` → PUSH | `[3]` |
| `3` | PUSH | `[3, 3]` |
| `-` | POP 3, POP 3 → `3-3=0` → PUSH | `[0]` |
| `4` | PUSH | `[0, 4]` |
| `2` | PUSH | `[0, 4, 2]` |
| `*` | POP 2, POP 4 → `4*2=8` → PUSH | `[0, 8]` |
| `+` | POP 8, POP 0 → `0+8=8` → PUSH | `[8]` |
| EOS | POP | 결과: 8 ✅ |

## 🎯 6차시 최종 암기 체크리스트

### 스택 기본
- [ ] LIFO, Top에서만 삽입·삭제
- [ ] 활용: Call Stack, Undo, 뒤로 가기, 수식 계산 ⭐
- [ ] `top` 초기값 = -1
- [ ] `peek = top` (삭제하지 않고 반환만)
- [ ] `is_empty`: `top == -1` / `is_full`: `top == MAX_SIZE - 1`
- [ ] 스택은 정적 구현 유리 (Shift 불필요)

### 정적 구현 PUSH/POP
- [ ] `++top`과 `top++` 차이
- [ ] PUSH: `stack[++top] = item;`
- [ ] POP: `item = stack[top--];`

### 동적 구현
- [ ] `top`이 가장 최근 삽입 노드를 가리킴
- [ ] PUSH: `new_node->link = top; top = new_node;`
- [ ] POP: `temp = top; top = top->link; free(temp);`

### 수식 계산 (시험 3문제!) ⭐
- [ ] 수작업 변환 3단계: 괄호 → 연산자 밖으로 → 괄호 제거
- [ ] 스택 변환: 피연산자는 출력, 연산자는 icp vs isp 비교
- [ ] isp < icp → PUSH, isp >= icp → POP 후 PUSH (같을 때도 POP 먼저!)
- [ ] `(` 는 들어올 때 icp 최고 / 들어간 후 isp 최저 (바닥에 깔림)
- [ ] `)` 만나면 `(` 나올 때까지 POP, 짝 괄호는 버림
- [ ] 중첩 괄호도 기계적 처리 (교수님 Q&A)
- [ ] 후위식 계산: 피연산자 PUSH, 연산자 만나면 2개 POP → 먼저 POP이 op2, 나중이 op1
- [ ] 성능: $O(n)$ / $O(n)$

### 지난 시간 복습 (빈칸 출제 주의!)
- [ ] 파이썬 `append`: Amortized $O(1)$
- [ ] 단순 LL `add_last`: $O(n)$
- [ ] 원형 LL `add_first`, `add_last`: 모두 $O(1)$
- [ ] 이중 원형 LL: 선행 노드 앞/뒤 모두 $O(1)$ (가장 많이 사용!)

### 실전 훈련
- [ ] `A / B * C` → `A B / C *`
- [ ] `A / B * (C + D) + E` → `A B / C D + * E +`
- [ ] `1 2 + 7 *` = 21
- [ ] `6 2 / 3 - 4 2 * +` = 8
- [ ] 스택 변환/계산 과정을 직접 빈 종이에 써보며 연습!
