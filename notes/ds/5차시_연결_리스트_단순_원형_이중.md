# 📘 5차시 — 연결 리스트 (단순 / 원형 / 이중)

> 핵심 한 줄 요약
> 연결 리스트는 자기참조 구조체(Node) 를 포인터(링크) 로 연결한 동적 자료구조. 삽입·삭제 시 데이터 이동이 없고 링크만 조작. 단순 → 원형(`addFirst`/`addLast`를 $O(1)$로) → 이중(양방향)으로 발전. 이중 연결 리스트 삽입/삭제에서만 8문제 출제 예고!

---

## 🎯 5차시 핵심 출제 포인트 프리뷰 (메모 기반)

1. 메모리 저장 영역: Global variable(Data area), Local variable(Static area), Dynamic data(Heap)
2. 원형/이중 연결 리스트 도입 배경: `addFirst`, `addLast`의 잦은 사용과 선행 노드 접근의 어려움 해결
3. 단순 연결 리스트 삽입의 3가지 경우: 공백 / 맨 처음 / 중간 — Head pointer 변경 여부 주의!
4. 삽입/삭제 시 포인터 연결 순서 (매우 중요): `before->rlink->llink` 등의 연결 순서 및 의미 이해
5. 💡 교수님 출제 예고: "이번 시간에 배운 단순/원형/이중 연결 리스트의 삽입과 삭제에서만 8문제가 출제됩니다. 다지선다형이므로 원칙만 알면 다 맞출 수 있습니다."

## 1. (복습) 정적(Static) vs 동적(Dynamic) 구현

| 항목 | 정적 (배열) | 동적 (연결 리스트) |
|------|------------|-------------------|
| 구현 방식 | C언어 배열, Python 리스트 | 자기참조 구조체 + 포인터 |
| 크기 / 타입 | 고정 (Python은 가변) | 가변 |
| 메모리 공간 | 연속 | 비연속 |
| 할당 시점 | Compile time | Run-time (실행 중) |
| 메모리 영역 | Data area (Global) / Static area (Local) | Heap (항상) |
| 탐색 | 인덱스 Random Access | Sequential Access (포인터 따라감) |
| 삽입/삭제 | $O(n)$ 데이터 이동 | 링크만 수정 (이동 없음) |
| 단점 | 메모리 낭비 + Shift 오버헤드 | 구현 복잡 + Segmentation Fault 위험 |

## 2. 포인터와 자기참조 구조체 심층 분석

### 2.1 포인터의 기본 개념

- 포인터: 데이터의 값(Value)이 아니라 주소(Address) 를 저장하는 변수.
- `&a`: 변수 `a`의 주소
- `*p`: 포인터 `p`가 가리키는 주소의 내용(Value)
- 데이터 타입의 중요성: 메모리에서 읽어올 크기(단위) 를 결정 (char 1B, int 4B, double 8B).
- 💡 디테일: 64비트 컴퓨터 기준 포인터 변수 자체의 크기는 8바이트.

### 2.2 🗣️ 교수님 Q&A 하이라이트

#### ① 포인터의 차원
- 최유타 학생의 질문: "포인터 별(`*`)은 몇 개까지 쓸 수 있나요?"
- 교수님 답변:
    - 문법적 제한 없음. 차원·계층(학생→반→학년→학교)을 나타낼 때 계속 늘릴 수 있음.
    - 하지만 실제 C 프로그램에서는 포인터의 포인터(``, 2개)** 까지만 주로 사용.
    - 3개 이상은 매우 드물며, Segmentation Fault 위험이 높아 권장 X.
    - 굳이 차원 높이기보다 2차원 배열·구조체 활용.

#### ② AI와 포인터 속도
- 학생 질문: "AI가 코딩하면 포인터를 엄청 깊게 써서 속도를 빠르게 할 수 있지 않나요?"
- 교수님 답변:
    - 포인터 많다고 속도 빠른 건 아님.
    - 탐색 가장 빠른 것은 인덱스 랜덤 액세스.
    - 연결 리스트는 순차적으로 따라가야 해서 느림.
    - 예: Numpy는 데이터 값을 연속적으로 직접 저장해 매우 빠르지만, 일반 Python 리스트는 객체의 주소를 저장해 상대적으로 느림.

### 2.3 자기참조 구조체 (Self-referential structure)

- 구조체 필드 중 자기 자신을 가리키는 포인터(링크 필드) 가 1개 이상 존재하는 구조체.
- 연결 리스트를 구성하는 핵심 단위인 노드(Node) 가 됨.
- 접근 연산자:
    - 구조체 변수 → `.` 연산자
    - 포인터 → `->` 연산자

```c
struct Listnode {
    int data;                   // 데이터 필드
    struct Listnode *link;      // 다음 노드를 가리키는 포인터 (자기참조)
};
```

## 3. 단순 연결 리스트 (Singular Linked List)

### 3.1 구조 및 특징

- 노드 구성: 데이터 필드(Data) + 링크 필드(Link)
- Head Pointer (헤드 포인터): 리스트의 첫 번째 노드를 가리키는 변수
- 마지막 노드의 링크 필드 값은 반드시 `NULL` — 이것으로 "단순 연결 리스트"임을 판별!

#### 시각화
```
head ──▶ [A | link] ──▶ [B | link] ──▶ [C | NULL]
```

---

### 3.2 💡 [출제 포인트] 삽입 연산의 3가지 경우

단순 연결 리스트에서 노드를 삽입할 때, head 포인터가 함수 안에서 변경될 수 있으므로 포인터의 포인터(`head`)** 를 매개변수로 받는다.

| 경우 | 상황 | Head 변경? |
|------|------|-----------|
| ① Head가 NULL | 공백 리스트에 삽입 — 새 노드가 첫 노드가 됨 | O (NULL → new_node) |
| ② 선행 노드(p) = NULL | 리스트 맨 처음에 삽입 (= `addFirst`) | O (기존 첫 노드 → new_node) |
| ③ 일반적인 경우 | 리스트 중간이나 끝에 삽입 | X (첫 노드 그대로) |

---

### 3.3 💡 [출제 포인트] 삽입 시 연결 순서 (매우 중요!)

교수님께서 "코드가 섞여서 출제될 수 있다" 고 강조한 부분.

#### 🔑 절대 원칙
> 삽입하고자 하는 새로운 노드(`new_node`)의 링크 필드를 먼저 설정해야 한다!

#### ⚠️ 이유
선행 노드(`before`)의 링크를 먼저 새 노드로 바꿔버리면, 원래 선행 노드가 가리키던 다음 노드(after)의 주소를 잃어버려 연결고리가 끊어진다.

#### ✅ 올바른 순서

```c
new_node->link = before->link;   // ① 새 노드가 다음 노드를 가리키게 함
before->link = new_node;         // ② 선행 노드가 새 노드를 가리키게 함
```

#### 시각화
```
전:  before ──▶ after

①:  new_node ──▶ after
     before ──▶ after   (아직 연결 유지)

②:  before ──▶ new_node ──▶ after   ✅
```

---

### 3.4 💡 [출제 포인트] 삭제 연산

삭제는 "공백 리스트(NULL)"인 경우를 고려하지 않는다 (아무것도 없는데 삭제 불가).

| 경우 | 상황 | Head 변경? | 코드 |
|------|------|-----------|------|
| ① 선행 노드(p) = NULL | 맨 앞의 노드 삭제 | O | `head = head->link;` (또는 `head = removed->link;`) |
| ② 선행 노드 ≠ NULL | 중간 노드 삭제 | X | `before->link = removed->link;` |

#### ⚠️ 교수님 강조 (메모리 누수 방지)
> 삭제 연산 후에는 반드시 `free(removed)` 를 통해 메모리를 반환해야 한다. 그렇지 않으면 운영체제(OS)에서 배운 Memory Leak 이 발생한다.

---

### 3.5 기타 연산 (순회, 병합, 역순)

| 연산 | 방법 |
|------|------|
| 순회 (Traversal) | 포인터 `p`를 헤드에 두고, `p = p->link`를 통해 `p == NULL`이 될 때까지 이동. (반복문 선호) |
| 병합 (Merge) | 첫 번째 리스트의 끝까지 이동한 뒤, 마지막 노드의 링크에 두 번째 리스트의 헤드를 연결 |
| 역순 (Reverse) | 3개의 포인터(`p`, `q`, `r`) 를 사용하여 링크 방향을 뒤집음 |

> 🗣️ 교수님 코멘트: "단순 연결 리스트에서 역방향으로 가기 위해 이렇게 복잡한 짓을 해야 합니다. 이것이 바로 이중 연결 리스트가 필요한 이유입니다."

## 4. 원형 연결 리스트 (Circular Linked List)

### 4.1 도입 배경 및 특징 (💡 메모 연계)

#### 단순 연결 리스트의 한계
- `addLast` (마지막에 삽입)를 하려면 항상 헤드부터 끝까지 순차 탐색($O(n)$) 해야 해서 비효율적.

#### 해결책
- 마지막 노드의 링크가 `NULL`이 아니라 첫 번째 노드를 가리키도록 만듦.

#### 💡 Head Pointer의 위치 (핵심!)
- 헤드 포인터가 첫 번째 노드가 아닌 마지막 노드(Last)를 가리키게 함.
    - `head` = 마지막 노드
    - `head->link` = 첫 번째 노드
- 이렇게 하면 `addFirst`와 `addLast` 모두 상수 시간 복잡도 $O(1)$ 로 처리 가능!

#### 시각화
```
           ┌──────────────────────┐
           ▼                      │
         [First] ──▶ [B] ──▶ [Last] ──┘
                                 ▲
                                head
```

#### OS 활용 예시
- 운영체제의 라운드 로빈(Round-Robin) 스케줄링 (시간 분할 프로세스 관리)에 원형 연결 리스트 개념이 사용됨.

---

### 4.2 원형 연결 리스트의 삽입

#### 원칙
- 항상 새 노드(`new_node`)의 링크 필드를 먼저 설정해야 한다.

#### 순서
1. `new_node->link = head->link;` (새 노드가 기존 첫 번째 노드를 가리킴)
2. `head->link = new_node;` (head의 링크를 새 노드로 변경)
3. 만약 마지막 노드로 삽입하는 경우: `head = new_node;` 로 헤드가 새로운 마지막 노드를 가리키게 업데이트.

## 5. 이중 연결 리스트 (Double Linked List) 🌟 [시험 출제 1순위]

### 5.1 도입 배경 및 특징 (💡 메모 연계)

#### 도입 배경
- 단순/원형 연결 리스트는 특정 노드에서 선행 노드(Previous Node)로 이동하려면 처음부터 다시 탐색해야 하는 치명적 단점이 있다.

#### 구조
- 노드에 `llink` (Left Link / Previous) 와 `rlink` (Right Link / Next) 두 개의 포인터를 두어 양방향 접근 가능.

```c
struct DNode {
    int data;
    struct DNode *llink;  // 이전 노드
    struct DNode *rlink;  // 다음 노드
};
```

#### 시각화
```
NULL ◀── [A] ◀──▶ [B] ◀──▶ [C] ──▶ NULL
```

#### 장단점

| 장점 | 단점 |
|------|------|
| 가운데 노드 하나만 알아도 양쪽 노드 모두 접근 가능 → 삽입/삭제 매우 용이 | 포인터 2개로 메모리 공간을 더 차지 |
| 양방향 순회 가능 | 코드가 복잡 |

#### 실제 활용
- 이중 원형 연결 리스트 (원형 + 이중) 형태가 가장 많이 사용됨.

---

### 5.2 💡 [핵심 출제 포인트] 삽입 연산 포인터 연결 순서 (p38, p40)

> 교수님께서 "시험 문제에 출제된다" 고 명시한 부분. 코드가 섞여서 출제될 때 순서를 정확히 맞춰야 한다.

#### 🔑 대원칙
> 삽입하려는 `new_node`의 `llink`와 `rlink`를 무조건 먼저 설정한다!

#### ✅ `before` 노드 다음에 `new_node`를 삽입하는 과정 (4단계)

```c
new_node->llink = before;              // ① 새 노드의 왼쪽은 before
new_node->rlink = before->rlink;       // ② 새 노드의 오른쪽은 before의 원래 다음 노드
before->rlink->llink = new_node;       // ③ 💡 before의 다음 노드의 llink를 새 노드로
before->rlink = new_node;              // ④ before의 오른쪽을 새 노드로
```

#### ⚠️ 순서가 중요한 이유
- ③에서 `before->rlink` 는 "before의 다음 노드"를 의미. 이 노드의 `llink`를 새 노드로 바꿔야 함.
- ④에서 `before->rlink`를 바꿔버리면 ③의 "before의 원래 다음 노드" 참조가 깨짐 → 반드시 ③ → ④ 순서!

#### 시각화
```
초기:        [before] ◀──▶ [after]

①:         new_node → llink = before
②:         new_node → rlink = after        (new_node가 after를 가리킴)

③:         before → rlink (=after)의 llink = new_node
           [before] ◀──▶ [after]
                    ┌──▶ [new_node] ──▶ after
                    └──  after.llink = new_node

④:         before → rlink = new_node
           최종: [before] ◀──▶ [new_node] ◀──▶ [after] ✅
```

---

### 5.3 💡 [핵심 출제 포인트] 삭제 연산 포인터 연결

삭제는 삽입보다 상대적으로 간단. 삭제할 노드(`removed`)의 양옆 노드들을 서로 직접 연결만 하면 된다.

```c
removed->llink->rlink = removed->rlink;   // ① 이전 노드의 rlink를 다음 노드로
removed->rlink->llink = removed->llink;   // ② 다음 노드의 llink를 이전 노드로
free(removed);                             // ③ ⚠️ 메모리 반환 필수!
```

#### 시각화
```
삭제 전:   [A] ◀──▶ [B: removed] ◀──▶ [C]

①:  A.rlink = C           (A → C)
②:  C.llink = A           (C → A)
③:  free(removed)          (B 메모리 반환)

결과:      [A] ◀──▶ [C] ✅
```

## 📝 5차시 종합 비교표

| 종류 | 링크 개수 | 마지막 노드 | Head 위치 | addFirst | addLast | 역방향 탐색 |
|------|----------|------------|-----------|---------|---------|------------|
| 단순 연결 리스트 | 1개 (link) | NULL | 첫 번째 노드 | $O(1)$ | $O(n)$ | 불가능 (재탐색 필요) |
| 원형 연결 리스트 | 1개 (link) | 첫 번째 노드 | 마지막 노드 | $O(1)$ | $O(1)$ | 불가능 |
| 이중 연결 리스트 | 2개 (llink, rlink) | NULL (또는 첫 노드) | 첫 노드 (또는 last) | $O(1)$ | $O(1)$ (이중 원형 시) | 가능 ⭐ |

## 🎯 5차시 최종 암기 체크리스트

### 단순 연결 리스트
- [ ] 노드 = 데이터 + 링크
- [ ] 마지막 노드 링크 = NULL
- [ ] 삽입 3가지 경우: 공백 / addFirst / 일반 (Head 변경 여부!)
- [ ] 삭제 2가지 경우: 맨 앞 / 중간 (공백은 고려 X)
- [ ] ⭐ 삽입 원칙: `new_node->link` 먼저 설정!
- [ ] `free(removed)` 필수 — Memory Leak 방지

### 원형 연결 리스트
- [ ] Head는 마지막 노드를 가리킴 (첫 번째 X!)
- [ ] `head->link` = 첫 번째 노드
- [ ] addFirst, addLast 둘 다 $O(1)$
- [ ] Round-Robin 스케줄링에 활용

### 이중 연결 리스트 🌟
- [ ] 노드 = 데이터 + llink + rlink
- [ ] 양방향 접근 가능
- [ ] ⭐ 삽입 4단계 순서 암기!
    1. `new_node->llink = before;`
    2. `new_node->rlink = before->rlink;`
    3. `before->rlink->llink = new_node;` 💡
    4. `before->rlink = new_node;`
- [ ] 삭제 3단계:
    1. `removed->llink->rlink = removed->rlink;`
    2. `removed->rlink->llink = removed->llink;`
    3. `free(removed);`

### 포인터 / 메모리
- [ ] `&`: 주소, `*`: 값
- [ ] 64비트 포인터 = 8바이트
- [ ] Heap = 동적 할당 영역
- [ ] 포인터의 포인터(`**`)까지만 실용, 3중 이상 위험
- [ ] `malloc` ↔ `free` 쌍 맞추기

### 시험 출제 예고
- [ ] 교수님: 이 단원에서만 8문제 출제!
- [ ] 코드 순서 섞여 나올 때: 주소 번지수 그려가며 풀기
