## 1. 자료구조의 구현 방식: 정적 vs 동적

모든 자료구조(선형·비선형 모두)는 크게 정적(Static) 또는 동적(Dynamic) 으로 구현 가능. 이 둘의 차이를 명확히 아는 것이 이번 파트의 핵심.

### 1.1 정적 구현 (Static Implementation)

#### 메모리 구조
- 연속된 기억 공간에 고정된 크기로 자료구조를 생성.

#### 언어별 특징
- C언어 (배열): 컴파일 타임(Compile time) 에 크기 확정. 예: `int A[100];`
    - 전역변수로 선언 시: Data area
    - 지역변수로 선언 시: Static area (스택/정적 영역)
- 파이썬 (List): 내부적으로 동적 배열(Dynamic Array) — 가변형(Mutable), 크기 미리 정할 필요 없음.

#### 장점
- 구현이 매우 간단 (선언만 하면 끝).
- 인덱스를 통한 랜덤 액세스 가능 → 탐색 매우 빠름.
- 내 다음에 오는 요소(Next element)의 위치를 쉽게 계산 가능 (예: `int`면 4바이트 뒤).

#### 단점
- 메모리 낭비: 100만 개 선언해놓고 100개만 쓰면 나머지는 버려짐.
- 💡 [핵심 출제] 데이터 이동(Data Shift) 오버헤드:
    - 선형 자료구조에서 삽입/삭제 시 빈 공간을 만들거나 채우기 위해 나머지 데이터를 전부 한 칸씩 이동.
    - 최악의 경우 $O(n)$ 의 선형 시간 복잡도.
    - (단, `append`는 상수 시간 $O(1)$)

---

### 1.2 동적 구현 (Dynamic Implementation)

#### 메모리 구조
- 비연속적인 기억 공간에 실행 도중(Runtime) 필요한 만큼(가변 크기) 생성.
- 힙(Heap) 메모리 사용.

#### 특징
- 데이터가 연속되어 있지 않으므로, 내 옆집(다음 데이터)을 찾아가려면 주소(Address) 를 알아야 함.
- 데이터뿐만 아니라 다음 엘리먼트를 가리킬 링크(주소) 필드가 반드시 필요.

#### 장점
- 필요한 만큼만 힙에서 할당 → 메모리 효율성 좋음.
- 삽입/삭제 시 데이터 이동 없음 (링크만 바꿈).

#### 단점
- 구현이 복잡.
- 인덱스로 한 번에 접근 불가, 순차적으로 포인터를 따라가야 함 (Sequential Access) → 탐색 속도는 배열보다 느림.

---

### 1.3 정적 vs 동적 한눈에 비교

| 항목 | 정적 (배열) | 동적 (연결 리스트) |
|------|------------|-------------------|
| 메모리 공간 | 연속적 | 비연속적 (Heap) |
| 크기 | 고정 (C), 가변 (Python) | 가변 |
| 할당 시점 | Compile time | Runtime |
| 탐색 | $O(1)$ Random Access | $O(n)$ Sequential Access |
| 삽입/삭제 (중간) | $O(n)$ Data Shift | $O(1)$ (링크만 변경) |
| 구현 난이도 | 쉬움 | 어려움 (포인터 필요) |

## 2. 💡 [시험 출제 포인트] 동적 구현의 핵심: C언어 포인터와 구조체

동적 구현(연결 리스트)을 이해하기 위해 교수님이 포인터와 구조체 개념을 상세히 설명.

### 2.1 포인터 (Pointer)

- 정의: 값이 아니라 주소(Address) 가 저장되는 변수.
- 데이터 타입의 중요성: 메모리에서 읽어올 크기(단위) 를 결정.
    - `char`: 1바이트, `int`: 4바이트, `double`: 8바이트
- 💡 64비트 컴퓨터에서 포인터 변수 자체의 크기: 8바이트.

#### 포인터 연산자

| 연산자 | 의미 |
|--------|------|
| `&a` | 변수 `a`의 주소(Address) 를 추출 (주소 연산자) |
| `*p` | 포인터 `p`가 가리키는 곳의 내용(Value) 을 추출 (참조 연산자) |

### 2.2 배열과 포인터의 관계 (매우 중요)

- 💡 1차원 배열의 이름은 그 자체로 "주소 상수(Address Constant)".
- 예: `int A[6];`에서 `A`는 배열의 첫 번째 요소의 주소 (예: 26번지)를 의미.
    - `A`는 주소, `A[3]`은 값(Value)
    - 따라서 포인터 `ptr`에 넣을 때는 `ptr = A;` 또는 `ptr = &A[3];` 형태로 써야 함.
- 배열 이름은 Base Address → `a`, `a+1`, `a+2`, ... 구조.

---

### 2.3 🗣️ 교수님 Q&A: 포인터의 차원과 속도

#### Q1. 최유타 학생: "포인터 별(`*`)은 몇 개까지 쓸 수 있나요?"
- 교수님 답변:
    - 문법적 제한 없음. 차원이나 계층(학생 → 반 → 학년 → 학교 ...)을 나타낼 때 계속 늘릴 수 있음.
    - 하지만 실제 C 프로그램에서는 포인터의 포인터(``, 2개)** 까지만 주로 사용.
    - 3개 이상은 매우 드묾. 주소를 조작하다 Segmentation Fault 발생 확률이 너무 높기 때문.
    - 굳이 차원을 높이기보다 2차원 배열·구조체 등 활용.

#### Q2. "AI가 코딩하면 포인터를 엄청 깊게 써서 속도를 빠르게 할 수 있지 않나요?"
- 교수님 답변:
    - 포인터 많다고 속도 빠른 건 아님.
    - 탐색 속도가 가장 빠른 것은 인덱스 랜덤 액세스.
    - 연결 리스트는 처음부터 순차적으로 따라가야 해서 느림.
    - 예: 파이썬의 Numpy는 데이터 값을 연속적으로 직접 저장하여 매우 빠르지만, 일반 파이썬 리스트는 객체의 주소를 저장하므로 상대적으로 느림.

#### Q3. "스트럭처 안에 포인터가 있으면 포인터의 포인터밖에 안 되나요? 세 단계는?"
- 교수님 답변:
    - 포인터 변수(`*p`)도 메모리 어딘가에 저장되어 있으므로 자신만의 주소가 있음.
    - 이 포인터 변수의 주소를 저장하는 것이 포인터의 포인터(`p`)**.
    - C언어는 강력한 권한을 주기 위해 이중 포인터를 제공.
    - 이론적으로 `*p`처럼 깊이 들어갈 수는 있지만, 루프(Loop)처럼 깊이 들어가면 Segmentation Fault 나 코드 꼬임 위험이 커서 권장하지 않음**.

---

### 2.4 구조체(Structure)와 자기참조 구조체

- 구조체: C언어에서 타입이 다른 데이터들을 하나로 묶는 방법. (배열은 같은 타입 묶음)
- 자기참조 구조체 (Self-referential structure) 💡:
    - 구조체 멤버 중 자기 자신과 같은 타입의 구조체를 가리키는 포인터가 한 개 이상 존재하는 구조체.
    - 연결 리스트(Linked List)를 만들 때 필수.
    - 접근 연산자: 구조체 변수.필드 / 포인터일 경우 `->` 연산자

#### 수도코드 핵심 메모

```c
struct Listnode {                 // 연결 리스트의 노드 정의
    int data;                     // 실제 데이터
    struct Listnode *link;        // 다음 노드를 가리키는 포인터 (자기참조)
};
```

---

### 2.5 동적 메모리 할당 함수 (malloc, free)

```c
ptr = (struct Listnode *) malloc(sizeof(struct Listnode));
```
- 실행 시간(Runtime)에 힙(Heap) 메모리에서 `Listnode` 크기만큼의 공간을 할당받고, 그 시작 주소를 `ptr`에 저장.

```c
free(ptr);
```
- 다 쓴 메모리는 반드시 운영체제에 반환해야 함 (안 하면 Memory Leak).

## 3. 참조 호출 (Call by Reference)의 3가지 케이스

> 왜 함수에 값이 아니라 주소(Reference) 를 넘길까? → 데이터 이동(복사) 오버헤드를 없애기 위해서.

| 케이스 | 설명 |
|--------|------|
| ① 함수의 파라미터 | Call by value는 데이터 전체가 복사되어 지역 변수가 생성되지만, 주소를 넘기면 복사 비용이 사라짐 |
| ② 연결 리스트 (Linked List) | 삽입/삭제 시 데이터 이동 없이 주소 링크만 변경 |
| ③ 배열(Array)을 함수에 넘길 때 | 1차원 배열 이름은 주소 상수이므로 배열 전체를 복사하지 않고 주소만 넘김 — 효율성 극대화 |

## 4. 추상 데이터 타입 (ADT)과 파이썬 내장 자료구조

### 4.1 ADT 복습
- 교수님 비유: "ADT는 자료구조의 사용 설명서다."
- 자료구조를 유지·관리하기 위해서는 객체에 대한 설명과 그 객체에 가해지는 연산이 명확히 정의되어야 함.
- 예: 스택은 스택의 방식, 큐는 큐의 방식대로 삽입/삭제해야 함. 큐 방식으로 삽입하면서 "스택"이라고 부를 수 없음.

### 4.2 파이썬의 Built-in 자료구조

#### 🔹 시퀀스(Sequence) 타입 — 순서(인덱스) 있음, 중복 허용

| 자료구조 | 특징 |
|----------|------|
| Tuple (튜플) | 반복 순회 가능, 불변형(Immutable) |
| List (리스트) | 반복 순회 가능, 동적 크기 조정 가능, 가변형(Mutable) |

#### 🔹 컬렉션(Collection) 타입

| 자료구조 | 특징 |
|----------|------|
| Set (세트) | 반복 순회 가능, 중복 없음, 인덱스·순서 없음 (정렬되지 않은 가변형) |
| Dict (딕셔너리) | 해시 테이블(Hash Table) 로 구현. Key-Value 매핑된 고유 항목 |

## 5. 선형 자료구조의 대표 선수: 리스트(List) ADT

리스트는 $n$개의 엘리먼트가 순서대로 1:1 관계를 가지는 모임. 스택/큐와 달리 처음·끝·중간 어느 위치에서나 삽입·삭제가 발생할 수 있다는 것이 가장 큰 특징.

### 5.1 리스트의 주요 연산 (Operations)

리스트에서는 특히 처음과 끝에서 연산이 매우 자주 일어난다.

| 연산 | 설명 |
|------|------|
| `add_last` | 리스트 맨 끝에 요소 추가 (매우 빈번, 파이썬의 `append`) |
| `add_first` | 리스트 맨 처음에 요소 추가 |
| `add(list, position, item)` | 임의의 위치(Position)에 요소 삽입 |
| `delete(list, position)` | 임의의 위치에 있는 요소 삭제 |
| `clear` | 모든 항목 삭제 |
| `replace` | 특정 위치의 항목을 다른 항목으로 변경 |
| `is_in_list` | 리스트에 특정 key값이 있는지 검색 |
| `get_entry` | 특정 인덱스의 엘리먼트 반환 |
| `get_length` | 리스트의 크기 반환 |
| `is_empty` / `is_full` | 비어/꽉 차있는지 체크 (삽입/삭제 전 필수 확인) |

---

## 6. 💡 [시험 출제 포인트] 정적 리스트(배열)의 삽입·삭제 메커니즘

> 교수님께서 "데이터의 이동! 별표는 함부로 하는 게 아니죠" 라고 농담 섞어 매우 강조한 부분. 정적 리스트 내부의 삽입·삭제 동작을 완벽히 이해해야 한다.

### 6.1 삽입 연산 `add(list, position, item)`

정적 공간에 데이터를 중간에 끼워 넣으려면 빈 공간을 만들어야 한다.

순서:

1. 사전 검사: `is_full` (꽉 찼는지), `position`이 유효한 범위인지 확인.
2. 💡 데이터 이동 (공간 만들기):
    - 중간에 칸을 만들기 위해 맨 마지막 요소부터 시작해서 삽입할 `position`까지 데이터를 뒤로 한 칸씩 이동(Shift).
    - ⚠️ 반드시 맨 뒤에서부터 역순으로 당겨야 한다! 앞에서부터 뒤로 밀면 데이터가 덮어씌워져 날아가 버림.
3. 삽입 및 마무리: 빈 공간이 된 `position`에 `item`을 넣고, 리스트의 `length`를 1 증가.

#### 시각화 (position=2에 `X` 삽입)
```
초기:      [A, B, C, D, E, _, _]   (length=5)
Step 1:   [A, B, C, D, _, E, _]   ← E 뒤로
Step 2:   [A, B, C, _, D, E, _]   ← D 뒤로
Step 3:   [A, B, _, C, D, E, _]   ← C 뒤로
Step 4:   [A, B, X, C, D, E, _]   ← position=2에 X 삽입
최종:      length = 6
```

---

### 6.2 삭제 연산 `delete(list, position)`

데이터를 지우고 나면 이빨이 빠진 것처럼 빈 공간이 생기므로, 이를 메워야 한다.

순서:

1. 사전 검사: `is_empty`, `position` 유효성 확인.
2. 항목 삭제: 해당 `position`의 아이템 제거(추출).
3. 💡 데이터 이동 (공간 채우기):
    - 빈 공간을 채우기 위해 삭제된 위치의 바로 다음(`position + 1`) 요소부터 맨 마지막 요소까지 앞으로 한 칸씩 이동.
4. 마무리: `length`를 1 감소.

#### 시각화 (position=2의 `C` 삭제)
```
초기:      [A, B, C, D, E, _, _]   (length=5)
삭제:      [A, B, _, D, E, _, _]   ← C 제거
Step 1:   [A, B, D, _, E, _, _]   ← D 앞으로
Step 2:   [A, B, D, E, _, _, _]   ← E 앞으로
최종:      length = 4
```

---

## 7. 파이썬 리스트의 내부 동작 (가변형 동적 배열)

파이썬의 리스트는 정적인 C언어 배열과 달리 크기를 지정하지 않고 계속 `append` 할 수 있다. 하지만 근본적으로는 배열 기반.

### 7.1 원리
- 내부적으로 일정 크기의 연속된 공간을 확보해 두고 사용.
- 꽉 차서 더 이상 삽입할 수 없게 되면, 알아서 메모리 공간을 더 크게(예: 두 배보다 조금 작은 비율) 새로 할당받음.

### 7.2 오버헤드
- 확장 시 기존의 모든 데이터를 새로운 넓은 공간으로 전부 복사(Copy) 한 뒤 새 값을 넣음.
- ⚠️ 따라서 파이썬 리스트 역시 근본적으로 연속된 공간을 쓰기 때문에 데이터 이동(Shift + Copy) 오버헤드를 피할 수 없음.

---

## 8. 다음 강의 예고 (연결 리스트)

정적 구현(배열, 파이썬 리스트)의 가장 큰 슬픔이자 한계는:
$$\boxed{\text{삽입·삭제 시 } O(n) \text{의 데이터 이동 오버헤드}}$$

연산의 횟수를 줄이는 것이 알고리즘 성능 분석의 핵심이므로, 이 데이터 이동을 없애기 위해 동적 구현 방식인 '연결 리스트(Linked List)' 가 등장한다.

> 🗣️ 교수님 코멘트: "다음 시간에는 포인터가 나와서 골치가 아플 수 있지만, 시험에 나오는 문제들이 굉장히 많이 나오는 시간입니다. 단순 연결 리스트, 원형 연결 리스트, 이중 연결 리스트의 개념을 확실히 잡아야 중간고사를 잘 볼 수 있습니다."

## 🎯 4차시 최종 암기 체크리스트

### 개념
- [ ] 정적 vs 동적: 할당 시점 (Compile vs Runtime), 메모리 영역 (Data/Static vs Heap), 연속성 (연속 vs 비연속)
- [ ] 정적: Random Access $O(1)$ 탐색, 삽입/삭제 $O(n)$ 데이터 이동
- [ ] 동적: Sequential Access $O(n)$ 탐색, 삽입/삭제 $O(1)$ 링크만 수정
- [ ] 64비트 시스템 포인터 변수 크기 = 8바이트
- [ ] 1차원 배열 이름 = 주소 상수
- [ ] 포인터의 포인터(`**`)는 실제로 많이 쓰지만 3중 이상은 위험

### ADT & 리스트
- [ ] 리스트: 어디서나 삽입/삭제 가능한 1:1 선형
- [ ] 주요 연산 10가지 암기
- [ ] 삽입 시: 뒤에서부터 역순으로 Shift (⚠️ 앞에서 밀면 데이터 날아감!)
- [ ] 삭제 시: `position+1`부터 앞으로 Shift
- [ ] 파이썬 리스트도 근본은 배열 → 확장 시 Copy 오버헤드

### Call by Reference 3케이스
- [ ] 함수 파라미터 / 연결 리스트 / 배열 전달

### 용어
| 용어 | 의미 |
|------|------|
| Static Allocation | Compile time, 연속 공간, 고정 크기 |
| Dynamic Allocation | Runtime, Heap, 가변 크기 |
| Self-referential structure | 자기 자신 타입 포인터를 가진 구조체 → 노드 |
| `malloc` / `free` | 힙 할당 / 반환 |
| Memory Leak | `free` 안 해서 발생 |
| Data Shift | 배열 삽입/삭제 시 데이터 이동 |
