## 🎯 시험 족집게 출제 포인트 총정리 (교수님 피셜)

### 🌟 2차시 (알고리즘 기초)
- 알고리즘의 5가지 조건 중 유한성(Finiteness) → 4지 선다형 1문제 확정
- 빅오(Big-O) 표기법의 정의·구하는 법·종류 → 100% 출제

### 🌟 3차시 (순환 vs 반복)
- 순환 4형태 (Linear / Non-tail / Tail / Binary)
- 순환 적합 조건 4가지
- 팩토리얼·피보나치·이진검색의 시간 복잡도 유도 과정
- "순환 함수의 수도코드를 주고 결과값을 묻는 문제" 출제 예고

### 🌟 5차시 (연결 리스트)
- "이번 시간에 배운 단순/원형/이중 연결 리스트의 삽입과 삭제에서만 8문제가 출제됩니다."
- 포인터 연결 순서 암기 (이중 연결 리스트 삽입 4단계)
- 원형 연결 리스트: head가 마지막 노드를 가리킴

### 🌟 6차시 (스택 & 수식)
- 수식 계산 파트에서만 3문제 출제!
- 중위식 → 후위식 수작업 변환 (객관식 4지선다)
- 스택을 이용한 변환 알고리즘: icp vs isp 비교 로직
- 후위식 계산: 피연산자 PUSH, 연산자 만나면 2개 POP

### 🌟 7차시 (큐 & 데크) — 중간 족집게 기출 3가지
- ① 스택 수식 변환/계산 트레이싱 (6차시 연계)
- ② 원형 큐 Empty/Full 조건
    - Empty: $\text{front} == \text{rear}$
    - Full: $(\text{rear} + 1) \% \text{MAX\_SIZE} == \text{front}$
- ③ 큐 연산 결과 예측 ("점수 드릴게요" 문제): `INQ INQ INQ DQ INQ DQ`

## 📝 핵심 개념 빠른 참조 (요약)

### 시간 복잡도 효율 순서
$$O(1) < O(\log n) < O(n) < O(n \log n) < O(n^2) < O(n^3) < O(2^n) < O(n!)$$

| 구분 | 범위 | 해결 가능성 |
|------|------|------------|
| Polynomial | $O(1) \sim O(n^3)$ (느슨히 $O(n^4)$) | 컴퓨터로 해결 가능 ✅ |
| Exponential | $O(2^n), O(n!)$ | 해결 불가 ⚠️ |

### 자료구조 특징 한눈에

| 자료구조 | 관계 | 원리 | 주요 응용 |
|----------|------|------|-----------|
| 리스트 (List) | 1:1 | 어디서나 삽입/삭제 | 범용 |
| 스택 (Stack) | 1:1 | LIFO (Top) | 함수 호출, 수식 계산, Undo |
| 큐 (Queue) | 1:1 | FIFO (Rear→Front) | 프린터 대기, OS 스케줄링 |
| 데크 (Deque) | 1:1 | 양방향 | — |
| 우선순위 큐 | 1:1 (실제 힙) | 우선순위 기반 | OS 스케줄링, 시뮬레이션 |
| 트리 (Tree) | 1:N | 계층 | 검색 트리, 힙 |
| 그래프 (Graph) | N:M | 연결/가중치 | 네트워크, 지도 |

### 대표 알고리즘 시간 복잡도

| 알고리즘 | 순환 | 반복 | 비고 |
|---------|------|------|------|
| 팩토리얼 | $O(n) + \alpha$ | $O(n)$ | 둘 다 OK |
| 피보나치 | $O(2^n) + \alpha$ ⚠️ | $O(n)$ | 반복 필수 |
| 이진 검색 | $O(\log n) + \alpha$ | $O(\log n)$ | 정렬된 데이터 필수 |

### 연결 리스트 주요 연산 복잡도

| 종류 | `add_first` | `add_last` | 역방향 탐색 |
|------|------------|-----------|------------|
| 파이썬 동적 배열 | $O(n)$ | Amortized $O(1)$ | — |
| 단순 LL | $O(1)$ | $O(n)$ | 불가능 |
| 원형 LL | $O(1)$ | $O(1)$ | 불가능 |
| 이중 LL | $O(1)$ | $O(n)$ | 가능 |
| 이중 원형 LL ⭐ | $O(1)$ | $O(1)$ | 가능 (가장 많이 사용) |

### 메모리 구조 (위에서 아래)
$$\text{OS} \to \text{Program} \to \text{Global/Static} \to \text{Heap (동적)} \to \text{Stack (함수)}$$

Stack Frame = Return Address + Local Variable + Parameter
